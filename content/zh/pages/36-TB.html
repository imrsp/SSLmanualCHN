对讲

SSL Live 控制台配有两个对讲通道，每个通道有两个输入和两类输出，因此可运行两套独立的对讲系统。例如，可将对讲通道 1 用于后台提示扬声器，将对讲通道 2 用于向乐手的入耳式监听发送讲话。

两个对讲通道均具有完整的通道控制和完整的信号处理功能。

基本的对讲控制位于主控区块右上角的 对讲 区域（如左图所示）。
旋转 TB1 和 TB2 编码器可改变对讲通道的推子电平；按下编码器可将相应对讲通道静音。

两类对讲输出概述如下：

对讲输出

主对讲输出用于馈送可能没有其他音频路由到达的远端扬声器或耳机，例如后台提示扬声器或另一位工程师的耳机。也可将其馈送到控制台的矩阵，以便向矩阵输出注入对讲信号。

主对讲输出使用主控区块 OUT 1 和 OUT 2 区域中的按键启用或关闭。 对讲 这些按钮也可在对讲通道的 Input/Routing 详细视图页面中使用（见下文）。

这些 OUT 硬件按键短按时会锁定开启；按住时则暂时启用，松开按键后自动关闭。

母线注入

也可将对讲信号发送到单独的 Aux 母线（见上图）、Stem 和 Master。向母线发送对讲信号时，使用相应母线 TB IN 中的按钮启用， Fader/TB 可在快速控制或详细视图页面中操作，详见本页后文。

启用对讲输入

每个对讲通道的两个输入均可在该通道的详细信息对话框中接入或断开：

将出现与下图类似的页面（如果显示的是其他对话框页面，请按 Input/Routing 按钮）：

两个输入的控制分别位于对话框两侧。每个输入都具有完整的输入配置控件，与输入通道 Input 详细信息对话框中的控件相同，相关介绍见 Detail View 章节。

此外，每个输入都有一个较大的 Input IN 按钮，用于接入或断开该输入（输入接入时按钮呈蓝色亮起）。借此可让一个或两个 TB 输入馈送至 TB 通道输出。

路由对讲输入和输出

控制台前缓冲区下方的 XLR 输入可用于对讲麦克风；左侧硬接线至本地输入 A15，右侧硬接线至本地输入 A16。在模板演出文件中，这两个输入分别路由至对讲通道 1 和 2 的 Input A。

对于 L300、L350、L450、L500、L500 Plus、L550 和 L650 控制台，对讲 XLR 输入还在内部与控制台后面板的 XLR 输入 A15 和 A16 并联硬接线。

注意： 如果相应的前缓冲区输入已经接入信号源，应避免再将信号源接入控制台背面的同一 XLR 输入，否则会降低音频性能。

路由在对讲通道的详细信息对话框（Input/Routing 页面）中完成。要更改 Input A 的信号源，请按 Route Input A 按钮，然后使用出现的标准路由对话框（详见 路由章节 ）。对于 Input B，请使用 Route Input B 按钮。主对讲输出（通过屏幕上的 TALKBACK OUT 按钮或主控区块 OUT 1/OUT 2 中的 对讲 按钮启用）使用 Route Output 按钮进行路由。
使用 Output Feed 按钮下方的 Route Output 按钮，选择主输出在信号路径中的取样位置： Post Trim, Pre Fader, Post Fader 或 Post All。这不会影响 TB 通道到 Aux/Stem/Master 的发送，这些发送始终取自 Post All 处理之后。

发送到母线

默认情况下，两个对讲通道都会路由到所有 Stem、Aux 和 Master 母线，发送增益设为 0 dB。

可使用对讲通道中的标准母线发送控件，通过快速控制、详细信息对话框或控制区块更改这些设置。

注意： 在控制台配置中新增的母线，其来自两个对讲通道的发送会自动切换为 IN，并提升到 0 dB。

随后可在各母线路径内启用对单独母线的对讲（也可通过对讲组启用；见 下文）。

要访问这些控件，请选择所需的 Aux/Stem/Master 路径，然后按通道视图中的 Fader/TB 分配按钮。

现在可按通道视图区屏幕推子左侧的 TB 按钮，或详细信息对话框 TB to Bus 中的 Fader/TB 按钮，以启用或停用对讲注入。

TB to Bus 也可从快速控制的 Fader 部分启用。按 Aux/Stem/Master 推子区块的快速控制编码器（或双击屏幕编码器），可在母线主推子控件和对讲控件之间切换快速控制页面。

另有 TB Dim 控件，可在 TB to Bus 启用时衰减主母线信号。每条母线都有各自的 TB Dim 电平，因此可根据艺人的偏好调整每个混音。

对讲可同时注入多条母线；只需为所需的 Aux/Stem/Master 启用 TB to Bus 。

单独的 TB to Bus 功能也可分配给用户按键，以便快速访问。更多信息请参阅 用户按键 选项页面。

TB to Bus 功能分配到硬件按键后，短按会锁定开启；按住则暂时启用，松开按键后自动关闭。

注意： 这些 TB to Bus 的启用状态是临时状态，不会存储到演出文件或场景中。这样可避免对讲麦克风意外保持开启并发送到母线。 TB Dim 电平会存储在演出文件中。

对讲组

系统提供十个对讲组，每个组可分配任意数量的母线，因此只需一个按钮或用户按键，即可将对讲注入多个 Aux/Stem/Master。

同一条母线可分配给多个对讲组。例如，歌手的 Aux 混音可同时包含在“Band”和“Backing Vocals”两个 TB 组中。任一已分配的对讲组开启时，该母线的对讲即会开启；所有已分配的对讲组均关闭后，对讲才会关闭。单独的 TB to Bus 功能在分配到对讲组后仍可用于切换单独的 TB 到母线发送。

要将母线分配给对讲组，请进入 Aux/Stem/Master 的 Fader/TB 详细视图页面，并在 TB Groups 部分启用十个按钮中的一个或多个。另请参阅下文所述的对讲组页面。

配置完成后，可将 TB 组分配给用户按键以便快速访问。更多信息请参阅 用户按键 选项页面。

TB Group 功能分配到硬件按键后，短按会锁定开启；按住则暂时启用，松开按键后自动关闭。

对讲组页面

这些 Talkback Groups 页面可用于查看、命名、分配和启用对讲组。访问路径为 MENU > Tools > Talkback Groups.

页面上半部分包含对讲组控件。按住 TB Group Assign 按钮进入分配模式。使用推子区块上的 Select 按键，将 Aux/Stem/Master 分配给所选对讲组。按所需范围的第一个和最后一个 Select 按键，可选择一段连续母线。再次按 TB Group Assign 按钮退出分配模式。

双击名称字段（默认为“Group N”），输入用户自定义的对讲组名称。将 TB 组分配给用户按键时会使用此名称。

名称字段下方列出分配给各 TB 组的所有母线。如果单个组分配了大量母线，可滑动此列表进行滚动。

另有 TB Group IN 按钮，可将对讲注入该组所列的全部母线。

页面下半部分包含所选路径的详细视图。与通道视图屏幕一样，可在此查看任一路径的处理和路由详情。
