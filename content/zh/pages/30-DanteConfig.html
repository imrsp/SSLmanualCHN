配置 Dante 设备和使用双域路由

双域路由

双域路由是 SSL 独有的功能，允许控制台直接与网络上的设备交互，无需使用 Dante Controller 来建立路由。

与普通的 Dante 路由一样，双域路由默认使用单播流创建。 如果需要组播流，则必须像往常一样在 Dante Controller 中设置。 来自这些发送通道的任何现有单播双域路由将自动转换为组播。随后从这些发送通道创建的任何双域路由都将是组播。

Dante 配置页面

在可以使用双域路由从控制台直接路由到 Dante 设备之前，必须在控制台的 Dante 配置页面中配置这些设备。

前往 MENU > Setup > I/O，然后点击菜单下方的 Dante Configuration。

页面左侧有一个 “Connected Devices”（已连接设备）列表。网络上的所有 Dante 设备都将在此列出。如果配置正确的设备未出现在列表中，请点击“Refresh Devices”按钮刷新视图。

页面右侧有一个 “Configured Devices”（已配置设备）列表。这是所有已配置供使用的 Dante 设备的列表。

“Configured Devices” 列表上方有两个资源条，用于指示可用的双域路由输入和输出总数以及当前使用的数量。 显示的总数将取决于采样率、虚拟跳线的数量以及是否使用了 BLII/X-Light 网桥。当创建双域路由时，资源条会填充浅蓝色或深蓝色：

浅蓝色：指向离线 “Configured Devices” 的路由（未连接到网络的设备，用于在舞台接口箱实际存在之前准备演出文件）

深蓝色：指向在线 “Configured Devices” 的路由（已连接到网络的设备）

每个设备都有一系列状态 LED：

有两个 NET LED，显示主备网络连接的状态。

颜色 含义
绿色 已连接
红色 未连接或故障

请注意：

当 SB 16.12 或 SB 32.24 连接了网络 A+B 链路时， 主备网络状态将始终报告两者都已连接，即使情况并非如此。 在此模式下，请查阅设备前面板上的 NET A 和 NET B LED 以了解主要和次要连接状态：

颜色 含义
绿色 Dante 主备网络已连接
红色 Dante 主网未连接或故障
闪烁红色 Dante 备网未连接或故障

更多信息请查阅 SB 16.12 和 SB 32.24 用户指南 (www.solidstatelogic.com/products/dante-live-sound)。

对于 “Configured Devices”，左侧有一个单独的 LED。这显示设备是在线还是离线。

颜色 含义
绿色 在线
红色 离线 - 设备之前曾在线（在此演出文件中）
灰色 离线 - 设备是离线创建的，并且从未在线（在此演出文件中）

对于 SSL Net I/O 舞台接口箱，还有 PSU 状态 LED。

颜色 含义
绿色 PSU 开启且功能正常
红色 PSU 关闭或故障

点击 “Connected Devices” 或 “Configured Devices” 列表中的任何设备都会调出该设备的详细视图。 如果设备是 “Connected Device”，则无法编辑舞台接口箱设置。如果设备是 “Configured Device”，则可以在此更改所有舞台接口箱设置和参数（例如增益、+48V、工作电平等等）。

以下信息显示在“Setup”选项卡中：

  • Dante 设备名称（应从 Dante Controller 更改，而不是从控制台更改）
  • I/O 容量
  • SSL 设备的 PSU 状态
  • 主网和备网网络状态
  • IP 地址（如果子网不一致，将显示为红色）
  • Dante 和控制台采样率（如果采样率不一致且控制台的 Dante SRC 关闭，将显示为红色）
  • 设备温度（如果可用）

请注意：

Dante 路由与设备名称相关联。 因此更改 Dante 设备名称将删除所有路由。重命名设备应在 Dante Controller 中进行，而不是在控制台上。 在更改设备名称之前，Dante Controller 中会出现一个警告对话框，通知用户音频路由将中断。

关于舞台接口箱的其他信息可以从右侧的相关选项卡中查看和编辑：Inputs、Outputs 等。 这些选项卡中列出的各个输入和输出通道名称将对应于 Dante Controller 中设置的通道标签。

配置在线 Dante 设备

现在我们将配置一个在线设备以进行双域路由。在线设备是连接到网络的设备。

首先，启用 Edit。 然后点击并拖动左侧 “Connected Devices” 列表中的一个设备。 一旦您开始拖动设备，“Configured Devices” 列表底部会出现一个 + 号。将设备拖放到 + 号上。

此设备现在已添加到 “Configured Devices” 列表中，您现在可以在路由视图或 XY 路由页面中对其进行双向路由。

配置离线 Dante 设备

即使物理设备不存在，也可以添加、配置离线设备并对其进行双向路由。 当设备可用时，这些设置随后可以与在线设备关联（见下文）。

首先，启用 Edit。然后点击“Configure New Device”并从列表中选择一个设备：

  • Net I/O SB 8.8*
  • Main (Uncompensated)
  • Compensated Splits
  • Net I/O SB i16*
  • Main (Uncompensated)
  • Compensated Splits
  • Net I/O SB 16.12*
  • Main (Uncompensated)
  • Compensated Splits
  • Net I/O SB 32.24*
  • Main (Uncompensated)
  • Compensated Splits
  • Net I/O A16.D16
  • Net I/O A32
  • Net I/O D64
  • Net I/O GPIO32
  • Net I/O MADI Bridge
  • SSL Console
  • Live Dante Expansion (带有 Dante 扩展模块的 SSL Live 控制台)
  • Net I/O BLII Bridge (连接到 BLII 网桥的 SSL Live 控制台)
  • Net I/O XL Bridge (连接到 X-Light 网桥的 SSL Live 控制台)
  • System T Dante HC (SSL System T 广播控制台系统)
  • System T Dante HC SRC (具有 SRC 功能的 SSL System T 广播控制台)
  • Shure
  • ANI22 (参见 Shure ANI 设备)
  • ANI4IN (参见 Shure ANI 设备)
  • Axient AD4D (参见 Shure 无线设备)
  • Axient AD4Q (参见 Shure 无线设备)
  • ULXD4D (参见 Shure 无线设备)
  • ULXD4Q (参见 Shure 无线设备)

选择您要添加的设备，一个离线设备将被添加到 “Configured Devices” 列表中。新设备的状态 LED 将为灰色，表示它从未在线。

您现在可以在任何路由视图（信号路径 Inputs/Outputs、FX Rack 等）或 XY Routing 页面中路由到/来自此设备。

建立双域路由

可以使用路由视图或 XY 路由页面以常规方式向任何已配置的 Dante 设备建立双域路由。 当向控制台上在线的已配置 Dante 设备建立双域路由时，您将在 Dante Controller 中看到在 Dante 网络上创建的路由。

重要提示：

请勿尝试从 Dante Controller 建立或取消双域路由。 所有双域路由都必须从控制台建立或取消。

将离线设备与在线设备关联

当离线 “Configured Device” 的设备名称和类型与 “Connected device” 匹配时，两者将自动关联。以前离线的设备将上线。 存储在离线设备中的所有舞台接口箱设置将被推送到舞台接口箱*，并且到/来自离线设备的路由将被分配给在线设备**。 这允许在断电或断开连接时不会丢失路由或设备设置。

*前提是满足所有所有权要求（参见 所有权 部分）

**前提是此设备与网络上其他设备之间的现有路由不存在路由冲突（参见 路由冲突）。

要关联离线和在线设备，首先点击 Edit，然后将在线的 “Connected Device” 拖放到相应的离线 “Configured device” 上。

点击 “Assign to Configured Device” 进行确认。

请注意：

设备类型必须相同才能生效。如果类型不同，控制台将不允许关联。

离线和在线设备现在已关联，离线设备的所有设置和路由都将分配给在线设备（前提是没有所有权或路由冲突）。

您也可以使用上述过程将现有的在线 “Configured Device” 与不同的在线“Connected Device”关联。

删除已配置设备

删除设备会将在线设备移回 “Connected Devices” 列表。控制台与此设备之间的所有路由将被取消。

要删除 “Configured Device”，请点击 Edit，使用右侧的圆圈选择 “Configured devices” 列表中的一个设备。然后按住 Delete。

取消关联

取消关联将在线设备与设备路由和设置分离，并将路由和设置保存在离线设备中。 取消关联设备会将在线设备移回 “Connected Devices” 列表。一个包含所有路由和设置的离线设备将保留在 “Configured Devices” 列表中。

要取消关联设备，请点击 Edit，使用右侧的圆圈选择 “Configured devices” 列表中的一个设备。然后按住 Unlink。

路由冲突

当控制台尝试向网络上的 Dante 设备建立路由，但已有一个 Dante 路由占用此路由点时，就会发生路由冲突。

这种情况可能在以下情况下发生：

  • 操作员（从控制台）手动在控制台和另一个 Dante 设备之间建立路由时
  • 加载包含 Dante 路由的演出文件时
  • 控制台开机且最近的演出文件包含 Dante 路由时
  • 将保存有路由的离线 “Configured Device” 关联到在线的 “Connected Device” 时

先到先得

当设备在网络上线或加载新的演出文件时，Dante 路由遵循先到先得的原则。 因此，如果到同一目的地的两条路由之间存在冲突，将保留最先创建的路由。 随后对同一目的地进行的任何路由都不会被建立。 这意味着 Dante 路由永远不会被自动取消。

但是，控制台可以在 XY Routing 页面中手动覆盖现有路由。请参见下面的 手动覆盖现有 Dante 路由。

当控制台断电或离线时

当控制台从主屏幕正确关机时，所有到/来自控制台的 Dante 路由都会被取消。 这释放了先前包含到/来自控制台路由的 Dante 设备， 这些路由点现在可以被网络上的其他 Dante 设备使用。

当控制台断电或未正确关机时（例如拔掉 PSU 电缆或关闭 PSU 电源）， 所有到/来自控制台的 Dante 路由都将保留。 这会导致控制台先前使用的其他 Dante 设备上的任何路由点无法用于进一步的路由。

注意：

出于此原因以及许多其他原因，SSL 建议始终通过主屏幕正确关闭 SSL Live 控制台 (MENU > Setup > System / Power menu > System tab > Power Off Console button)。 在从主屏幕正确关机之前，请勿拔掉 PSU 电缆或关闭 PSU 电源。

如果控制台与 Dante 网络的连接断开并离线， 网络上的设备将保留所有 Dante 路由。因此，当控制台重新上线时，无需重新建立路由。

当控制台上线或加载演出文件时

加载包含双域路由的演出文件可能会与网络上已有的路由发生路由冲突。 当控制台连接到网络或开机并自动加载最近的演出文件时，也可能发生这种情况。

例如：

一个 L550、一个 L200 和一个 SB 32.24 在同一个 Dante 网络上。 SB 32.24 输入 1 被路由到 L550 上通道 1 的输入。 并且通道 1 的输出被路由到 SB 32.24 输出 1。 L200 连接到网络，但在当前的演出文件中，控制台和 SB 32.24 之间没有路由。

L200 上加载了一个新的演出文件。在新的演出文件中，SB 32.24 输入 1 被路由到 L200 上的通道 32。 并且该通道的输出被路由到 SB 32.24 上的输出 1。

对于 SB 32.24 的发送通道，这没有问题。 SB 32.24 的输入 1 可以同时发送到 L550 和 L200。

对于 SB 32.24 的接收通道，存在路由冲突。 SB 32.24 不能同时接收来自 L550 和 L200 的路由。

根据 Dante 路由的先到先得原则，L550 的路由将被保留。 这是因为在 L200 尝试建立路由时，L550 的路由已经有效。 L200 的路由将不会被建立。

这是为了避免设备上线时“窃取”活动路由。

注意：先到先得原则是 Dante 的概念，同样适用于 SSL 和非 SSL Dante 设备。

如果将 “Configured Device” 分配给一个具有现有路由的新的“Connected Device”，则同样的先到先得原则适用。 要检查网络路由是否已创建，请使用 Dante Controller 的 Routing 选项卡。

手动覆盖现有 Dante 路由

Dante 路由永远不会被自动取消。但它们可以被控制台手动覆盖。在 XY Routing 页面中，工程师可以通过创建到同一目的地的新路由来覆盖现有路由。 这在 Routing View 中是不可能的。

例如：

一个 L550、一个 L200 和一个 SB 32.24 在同一个 Dante 网络上。 L550 上通道 1 的输出被路由到 SB 32.24 输出 1。

在 L200 上，工程师前往 XY Routing 并尝试建立从 L200 上通道 2 的输出到 SB 32.24 输出 1 的路由。

这导致了路由冲突。 SB 32.24 不能同时接收来自 L550 和 L200 的路由。

由于 L200 是手动尝试路由（即路由是工程师的选择）， L200 可以覆盖 L550 和 SB 32.24 之间的现有路由。 来自 L550 的路由将被取消，而来自 L200 的路由将被建立。

其他两个 Dante 设备之间的 Dante 路由

网络上的任何其他两个 Dante 设备之间的路由也可以从 XY Routing 页面手动建立或取消。 如果从控制台建立了其他两个 Dante 设备之间（即不涉及控制台）的路由，则该路由将保存在控制台的演出文件中。 当在控制台上加载演出文件时，该路由将被调用（前提是具有相同 Dante 名称的设备在网络中，并且遵守任何路由冲突，请参阅上面的 路由冲突）。

Dante 允许设备列表

Dante 和 AoIP 的优点和挑战在于可以自由地连接任何地方的任何设备。SSL Live 通过将 Dante API 与自动设备发现相结合来解决这个问题，提供了一个简单的拖放界面，用于将发现的设备添加到控制台的配置中，以简化路由。对于较大网络上的固定安装，SSL Live 的 “Dante Allowed Device List” 允许您管理哪些 Dante 设备在控制台上可见和可路由，非常适合自由工程师访问控制台但某些资源必须保持受限的固定安装。

此功能在获得 Dante Allowed Device List 许可证后启用，可将可见设备限制为仅 “AllowedDevicesConfig.xml” 文件中列出的设备，如果未获得许可，则保持所有标准控制台功能。支持 SSL 网络 I/O 和第三方设备，虽然 Rx 订阅仍可通过 Dante Controller 管理，但 SSL 界面会在路由 GUI 中使用带阴影的棕色覆盖层标记受限的 Rx 通道，防止路由更改或演出文件覆盖。如果您有 Dante Allowed Device List 的使用案例，请与您的 SSL 销售或支持代表讨论工作流程和许可事宜。

Dante Allowed Device List 可确保网络资源的安全，确保只能访问选定的设备和通道。非常适合大型、灵活的音频网络。

允许设备列表基于设备名称而不是 Dante 设备 UID。这具有以下含义：

  • 可以离线（在控制台或 SOLSA 上）配置（和命名）设备并将路由存储在演出文件中，以后可以在检测到正确名称的允许设备时将这些路由播放到网络上。
  • 更改 Dante 网络上设备的名称将需要在利用此功能的每个控制台上更新允许列表（需要访问服务页面）。
  • 可以在 Dante Controller（或其他具有控制功能的界面）中重命名设备，以匹配允许列表中存在的设备名称，从而启用该设备在控制台上的使用（受允许列表中定义的受限 Rx 通道约束），而无需修改允许列表。
