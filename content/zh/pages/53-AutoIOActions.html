章节：混音 | 来源： https://livehelp.solidstatelogic.com/Help/AutoIOActions.html

LIVE 的自动化 Input 和 Output Actions 可让场景由远程触发，也可让控制台控制外部事件，例如外接效果器的跳线切换。

输入动作

Input Actions 可通过控制台的 Event Manager 或外部事件触发场景；外部事件可以是 GPI（通用输入）或指定 MIDI 消息。

如需为场景启用 Input Action 触发，请触摸场景列表中 Input Action 方框之一；这些方框位于 Scene Name 左侧。方框会突出显示（变为青色）。（也可触摸 Automation 详细信息对话框左侧的 Input Actions 按钮，打开该动作的配置页面。）

然后触摸 TYPE 标签下方的按钮，并从子菜单中选择 GPIO、 MIDI 或 Event 。

请注意： L100 没有本地 GPIO 或 MIDI 端口；L100 只能配置使用 ipMIDI 或远程舞台接口箱 GPIO/MIDI 的 Input/Output Actions。
GPIO 输入动作

选择 GPIO 时，Input Actions 的 TYPE 列表会显示类似如下的详细信息对话框：

按 Apply 启用 Route Input 按钮。按下 Route Input 会打开路由菜单，显示所有可用 GPI 信号源。

选择其中一个信号源并按 Make ，即可将该 GPI 引脚分配给输入动作的第一个触发器。也可对整组 GPI 连接执行 Make All ，将每个 GPI 引脚分别路由到一个触发器。

GUI 主区域中的十二个分区即为触发器。如果只路由了一个 GPI 引脚，则只有第一个触发器有效。每个触发器有两个按钮：

左侧按钮定义触发动作的输入跳变类型：上升沿或下降沿。

右侧按钮用于将各触发器切换为 In。一个 Input Action 可将多个 GPI 引脚切换为 In ，这样任意一个 任意 已路由的 GPI 信号源都可触发场景。

MIDI 输入动作

选择 MIDI 时，Input Actions 的 TYPE 列表会显示类似下图的详细信息对话框：

首先选择要用于触发场景的 MIDI 消息类型： Note On、 Note Off、 PRG Change （Program Change）或 CTRL Change （Control Change，也称 MIDI CC 消息）。选择 Note On 会显示上图页面，并提供以下选项：

  • All Channels 按钮：启用时，监听所有 MIDI 通道上符合所定义参数的消息。禁用此按钮后，会增加另一个数值字段，用于输入 1 至 16 之间的指定 Channel 编号。
  • Note Number：0 至 127 之间、表示音符的数值（Note Number 60 = C3）。该值会自动更新下述 Note Name 字段。
  • Note Name：音名（C、C#、D 等）和八度编号（受上述 Middle C 数值限制）。该值会自动更新上述 Note Number 字段。
  • Velocity：音符力度值，范围为 1 至 127。

选择 Note Off 会显示与 Note On相同的选项，但不包含 Velocity （力度） 参数。

选择 PRG Change 后，可指定范围为 1 至 128 的 Program Number（音色号） ，并选择监听 All Channels 或指定的 MIDI 通道编号。

选择 CTRL Change 后，可指定范围为 0 至 119 的 Control Number（CC号） 、范围为 0 至 127 的 Control Value（C值） ，并选择监听 All Channels 或指定的 MIDI 通道编号。

注意： 如果使用 ipMIDI 传输 MIDI Input Actions，应使用不同于以下功能所用端口的 ipMIDI 端口： Timecode、 DAW Control 和 Events。
Event 输入动作

选择 Event 时，Input Actions 的 TYPE 列表会显示类似下图的详细信息对话框：

从 LINKABLE EVENTS 列表中选择事件。只有以 Automation Output 作为事件源的事件才会出现在此列表中。

触摸 Apply 应用配置。场景列表中的 Input Action 方框会增加一个图标，表示该场景可由 Input Action 触发。

重要说明： 按下 Apply后，必须为传入的 MIDI/GPI 数据选择信号源。该信号源可以是操作面背面的本地端口（如已安装）、舞台接口箱上的远程端口，或通过 Connectivity 网络端口传输的 ipMIDI（配置说明请参阅 ipMIDI 章节 ）。按 Route Input 按钮打开相应对话框。从列表中选择信号源并按 Make。

按 Arm ，允许 Input Action 触发该场景。

如果希望 Input Action 触发后仍保持待命，请保持 One Shot 未启用；如果希望 Input Action 触发一次后解除待命，请启用 One Shot 按钮。这样，只有第一次满足 Input Action 条件时才会触发场景。

现在可为该场景或其他场景配置其他 Input Actions。每个场景最多可定义三个 Input Actions。

可使用 Automation Arm All Input Actions 和 Disarm All Input Actions 按钮批量将所有场景中的全部 Input Actions 设为待命/解除待命；这些按钮位于 Controls 页面。也可使用 Event Manager触发这些动作。

如需从场景中移除 Input Action，请选择 None ，即 Input Actions TYPE 列表中的该项，然后按 Apply。

注意： Input Actions 按“或”逻辑工作：任意一个 Input Action 被触发时都会切换场景。一个 Input Action 中也可启用多个 GPIO 输入，同样按“或”逻辑工作。

输出动作

Output Actions 可在触发场景时让控制台向外部设备发送消息；消息可通过 GPO（通用输出）或指定 MIDI 消息发送。

如需在场景中创建由 GPO、MIDI 或 Event 驱动的触发动作，请触摸场景列表中 Output Actions 方框之一；这些方框位于 Scene Name 右侧。方框会突出显示（变为青色）。（也可触摸 Automation 详细信息对话框左侧的 Output Actions 按钮，打开该动作的配置页面。）

触摸 TYPE 标签下方的按钮选择动作类型，并从子菜单中选择 GPIO、 MIDI 或 Event 。

动作类型按钮下方的 At 按钮用于选择相对于场景的哪个时间点触发 Output Action：

  • At Fade Start ：场景刚刚触发时，即场景最开始
  • At Fade End ：所有参数完成淡入/淡出后
  • At Scene End ：离开场景时，即触发下一场景时
GPIO 输出动作

该 GPIO Output Actions 详细信息对话框与 GPIO Input Actions 页面（见 上文）相似，但每个 GPO 还增加了 Test 按钮。

路由方式相同。只路由一个 GPO 时，只有第一个触发器有效。使用 Make All 按钮后，每个触发器分别对应一个已路由的 GPO 引脚。

左下按钮定义触发的 GPO 类型：上升沿、下降沿、正脉冲或“负”脉冲（+12V -> 0V -> +12V）。按钮角落的绿色指示灯在 GPO 设为 +12V 时显示绿色，在 0V 时显示灰色。

右下按钮用于将各 GPO 引脚切换为 In。将多个 GPO 引脚全部切换为 In，即可让一个 Output Action 启用多个 GPO 引脚。

该 Test 按钮可切换 GPO 引脚上的电压，用于检查输出是否正常工作。

MIDI 输出动作

该 MIDI Output Actions 详细信息对话框与 MIDI Input Actions 页面（见 上文）相似，并为每种 MIDI 消息类型提供相同参数：

  • Note On 消息包含 Note Number（音符号） （0-127）、 Note Name（音名） 和 Velocity（力度） （1-127）参数。按钮启用时可在 All Channels 上发送；按钮禁用时可在指定 Channel （1-16）上发送。
  • Note Off 消息包含 Note Number（音符号） （0-127）和 Note Name（音名） 参数。按钮启用时可在 All Channels 上发送；按钮禁用时可在指定 Channel （1-16）上发送。
  • PGM Change 消息只包含 Channel 编号和 Program Number（音色号） （1-128）参数。
  • CTRL Change 消息包含 Channel 编号、 Control Number（CC号） （0-119）和 Control Value（C值） （0-127）参数。
Event 输出动作

该 Event Output Actions 详细信息对话框与 Event Input Actions 页面（见 上文）相似。

从 LINKABLE EVENTS 列表中选择事件。只有以 Automation Input 作为事件目标的事件才会出现在此列表中。

触摸 Apply 应用配置。

重要说明： 按下 Apply后，必须为 MIDI/GPO 数据选择目标。该目标可以是操作面背面的本地端口（如已安装）、舞台接口箱上的远程端口，或通过 Connectivity 网络端口传输的 ipMIDI（配置说明请参阅 ipMIDI 章节 ）。按 Route Output 按钮打开相应对话框。从列表中选择目标并按 Make。

按 Arm 按钮，允许该 Output Action 随此场景触发。

如果希望每次调用场景时都触发 Output Action，请保持 One Shot 未启用；如果希望 Output Action 触发一次后解除待命，请启用 One Shot 按钮。这样，只有第一次调用场景时才会发送 Output Action。

现在可为该场景或其他场景配置其他 Output Actions。每个场景最多可定义九个 Output Actions，触发场景时会将其全部触发。

可使用 Automation Arm All Output Actions 和 Disarm All Output Actions 按钮批量将所有场景中的全部 Output Actions 设为待命/解除待命；这些按钮位于 Controls 页面。也可使用 Event Manager触发这些动作。
