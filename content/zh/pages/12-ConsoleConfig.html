控制台配置

SSL Live 具有高度可配置性，可针对每场演出彻底重构。控制台提供 DSP 信号路径池，作为设计和构建控制台的基础；每条路径均可设为输入通道、Stem 编组、Aux 辅助母线或 Master 主母线。立体声、LCR、4.0 和 5.1 格式通过绑定两条或更多 DSP 路径创建。

信号路径类型简介

完整路径与干路径

所有 SSL Live 控制台都具有完整路径，可使用通道内全部信号处理功能（EQ、动态、延迟等）。

SSL Live 控制台还具有干路径，除输入增益与增益微调、推子、静音、独奏和两个插入点外，不包含其他信号处理。

注意：干路径虽然没有内置信号处理，但具有插入路由，并可使用 SSL Live 内部效果器机架进行处理。

信号路径类型

主要有 4 种路径类型：

  • 通道
  • Stem 编组母线
  • Aux 辅助母线
  • Master 主母线

本节稍后会详细介绍每种路径类型。

另有一些独立于主路径池的信号路径，包括 VCA、Talkback、Solo 和 Matrix 通道。它们均可通过下文所述的Console Configuration页面配置。

控制台还包含不执行信号处理、仅用于控制外部设备的路径：DAW paths（按需自动创建）以及OSC paths（通过 Console Configuration 页面创建）。

请注意：每种控制台型号都有可用路径总数限制，各路径类型（完整路径/干路径及通道/Stem/Aux/Master）也分别有限制。满足以下任一条件时，控制台会阻止继续创建路径：
- 已达到控制台路径总数上限。
- 已达到某种路径类型的上限。此时仅禁止继续创建该类型路径。

Plus 处理扩展包

L100、L200、L350 和 L550 可通过增加 Plus 处理扩展包扩展 DSP 容量。此功能需要许可证；详情请联系当地 SSL 经销商。启用后，控制台即为 Plus 版本，状态栏中会显示符号。其余功能与非 Plus 版本完全相同。因此除非另有说明，文中提及这些型号时同时包含 Plus 与非 Plus 版本。

Stem 编组母线

Stem 路径用于效果发送、共享信号处理、并行处理、音频子编组等。

由于拥有独立的源发送电平，它兼具传统音频编组母线和Aux 发送的功能，可视为功能最全面的音频母线。

Stem 的优势在于灵活的发送、母线分配和路由。Stem 可发送到 Aux 和其他 Stem，可汇入 Master，也可路由到 Matrix 通道或直接输出。

Stem 的馈送点明显多于 Aux，包括推子前、Trim 后、插入点 A 后、插入点 B 后、推子后和全处理后。

提示：每条路径中的处理模块顺序（EQ、压缩、门限等）均可自由更改，包括两个插入点。详情参阅处理顺序。
结合 Stem 可充分利用此功能。例如，将同一人声话筒通道的压缩前和压缩后信号分别发送到歌手自己的 IEM 混音（通过 Stem 1）及其他表演者的混音（通过 Stem 2）。

Stem 与对应的通道路径有以下区别：

  • 可以从通道、Stem 和 Aux发送到Stem。
  • 它没有外部输入路由对话框，因为信号通过母线送入，而不是通过路由送入。
  • Stem Query会收集所有馈入它以及由它馈送的路径，而不只是它所馈送的路径。详情参阅Query。

使用 Stem 进行监听混音

Stem 可作为 Aux 输出前的中间母线来管理监听混音，例如：

  1. 为经常成组使用的通道创建 Stem，例如鼓、打击乐、铜管等。当艺人要求增减整组声音时，只需在其 Aux 混音中调整 Stem，而不必逐个调整通道。不同混音可采用不同 Stem；例如鼓手混音可继续使用各鼓通道，而和声歌手混音可使用预混的“鼓”Stem。
  2. 如果多位乐手共用一个 IEM 接收包，可将 Aux 路径视为接收包而非乐手混音。改在 Stem 上创建混音，并将相应 Stem 取消静音后送往该接收包。这样可建立混音库并动态切换到 Aux 混音，甚至可用自动化系统自动切换。
  3. 播放技术人员通常需要监听未经处理的干播放轨，因此不希望听到路径处理。可将播放通道通过Post Trim汇入 Stem，再与其他来源（喊话系统、对讲等）一起送入其 Aux 混音。

Aux 辅助母线

Aux 路径用于需要独立混音的输出，例如艺人监听、通信或超低音发送。它也可作为传统效果发送，但 Stem 往往更适合，因为 Stem 可直接路由到其他 Aux。

Aux 路径与输入通道有以下区别：

  • 只能从通道和 Stem 向其发送；没有输入源路由对话框。
  • 不能发送到 Stem 或其他 Aux，但可汇入 Master，或路由到输出或 Matrix。
  • Aux Query会收集馈入它以及由它馈送的路径，而不只是由它馈送的路径。详情参阅Query。

Aux 可从源路径（Channel 或 Stem）的推子前或全处理后处理点馈送。

Master 主母线

Master 路径是混音的最终母线。所有汇入 Master 的信号均为推子后信号。Master 输出通常路由到矩阵或直接路由到外部目标。

Master 路径与对应 Channel 路径有以下区别：

  • 只能从其他路径通过母线馈入；没有输入源路由对话框。

  • 作为信号链最后一级，不能发送到 Stem、Aux 或其他 Master。

  • Master Query只收集所有馈入它的路径。详情参阅Query。

VCA

VCA 可通过一条控制路径控制多条路径的输出及母线发送电平。其推子条与其他路径一样显示在操作面上。Channel View 条顶部仪表区域会改为显示该 VCA 所包含的全部路径，并按路径类型列出。
可使用路径的 Mute Safe 功能使其不受 VCA 静音控制，详见Mute章节。

将路径分配给 VCA 与发送到母线非常相似，两项功能均在母线路由与 VCA章节中说明。

按下 VCA 的Solo按钮会独奏该 VCA 控制的所有路径，效果等同于分别按下每条路径的Solo按钮。

注意：移动 VCA 推子不会使所分配路径的推子移动，但这些路径上的推子增益标签会随 VCA 值变化。

控制台配置页面

可在以下位置编辑每种路径类型的数量：MENU > Setup > Console Configuration。

此处用于将可用数字信号处理资源分配给不同路径类型。可设置各种路径格式的准确顺序，也可设置 Aux 和 Stem 取源所用的路径馈送点；还可在此界面为路径命名。

该Console Configuration页面主体是一张表格，显示每种类型和格式的路径数量：

屏幕顶部资源条显示完整处理路径、干路径及各资源池路径类型的可用 DSP。控制台可在各自上限内任意组合这些路径，不会发生过载；达到上限时路径数量会变红，且无法继续创建。

各路径类型的完整处理形式列在屏幕上部，干路径形式列在其下；VCA、OSC、Matrix 和 Solo 通道列在底部。

注意：VCA、OSC、Matrix 和 Solo 通道的可配置项较少。此界面中没有可配置的 Talkback 项，因此不会显示；矩阵将在本页后文介绍。

输入通道、Aux、Stem 和 Master 可配置为单声道、立体声、LCR、4.0 或 5.1，各格式和类型数量按列显示。单声道使用一条 DSP 路径，立体声使用两条，LCR 使用三条，4.0 使用四条，5.1 使用六条。

每行右侧各列显示该类型路径数量（每条格式化路径计为一条）、已用 DSP 路径总数及该类型可用 DSP 路径上限。

添加/删除路径

要更改配置，请点击页面右侧的Edit按钮；页面可编辑时按钮会变蓝。

所有可编辑数字都会显示在深色框中。

双击数字，使用弹出的数字键盘修改，然后点击OK。

注意：如果留空或输入非数字字符，框会显示红色边线；必须更正后才能继续。

Stem、Aux 和 Master 可从处理路径内不同馈送点取源。每条母线可设默认馈送点，并可由各路径馈送点覆盖（详见母线路由）。可选默认馈送点如下：

  • Stem：Post Trim、Pre Fader、Post Insert A、Post Insert B、Post Fader 或 Post All 处理点
  • Aux：Pre Fader 或 Post All 处理点（包括推子）
  • Master：Post Fader 或 Post All 处理点

点击各类型 View 列中的箭头，可展开 Aux 和 Master 行，分别显示两个馈送点选项。展开后的数字可按常规方式编辑。再次点击 View 向下箭头可折叠该类型行。每条 Stem 的馈送点则显示在展开的列表视图中（见下文）。

注意：如果添加母线路径时未指定馈送点，Stem 和 Master 将创建为Post All，Aux 将创建为Pre Fader。

路径编号按其通过单声道、立体声、LCR、4.0 和 5.1 数字框添加的顺序分配；也可逐条编辑和添加各类型路径。

为此，请点击列表图标，它位于View列中。（Aux 和 Master 只有在展开视图中才显示此图标。）

相关类型的每条路径会单独显示一行，绿灯表示当前格式。点击所需格式的指示器使其变绿。已更改路径会显示 ♦ 符号。

每个展开列表底行还可按所选格式在列表末尾添加新路径。新路径会显示 ♦ 符号。

按住Apply按钮（右上角）应用更改。

请注意：离开Console Configuration页面，或未按Edit就关闭Apply，都会取消全部更改。

矩阵配置

SSL Live 有四个矩阵。它们可将可见交叉点缩减为最可能使用的部分，聚焦完整矩阵中相关区域，无需浏览大面积无用控制项。

要配置四个矩阵，请点击显示区 Matrix 区域 View 列中的列表图标。

矩阵输入和输出上限如下：

每个矩阵最多可有 32 个输入，不受其他矩阵配置影响。

四个矩阵合计最多有 36 个输出（因型号而异），可按任意方式分配。

注意：矩阵输出通道具有与完整处理路径相同的 Filters、EQ、Delay 和 All Pass 处理，但没有动态处理。

更多矩阵配置说明见矩阵页面。

按住Apply按钮（右上角），使 Console Configuration 页面中的更改在控制台上生效。

删除、移动和复制路径

可在路径列表中删除、移动或复制单条路径。

展开路径列表并启用Edit后，点击相关路径Edit列中的灰色单选按钮；每条选中路径的按钮会变红，表示正在编辑。

注意：可同时标记多条路径进行编辑。
  • 要删除路径，请按住Delete按钮。
  • 要移动路径，点击Move，再点击希望移动路径出现在其后的路径。
  • 要复制路径，点击Copy，再点击希望副本出现在其后的路径。
    复制路径的名称由短横线组成；双击名称字段可重命名。
    启用Apply功能后会刷新路径 ID 列。
    如果可用处理资源不足以创建所选数量的副本，Copy按钮会禁用。
注意：这些操作会作用于最近选择的路径类型区域中所有选为 Edit 的路径，无论该区域是否展开，也无论操作时路径是否可见。

按住右上角的 Apply 按钮，使 Console Configuration 页面中的更改在控制台上生效。

编辑路径名称

展开路径列表并启用Edit后，双击名称文字打开屏幕键盘。输入新名称并点击OK确认。

使用键盘中的Next和Previous按钮（USB 键盘分别为 Tab 和 Shift+Tab）在路径列表中导航。

再次点击列表图标可将路径折叠回路径类型行。

按住右上角的 Apply 按钮，使 Console Configuration 页面中的更改在控制台上生效。

导入/导出通道名称

SSL Live 控制台可从 CSV 文件导入输入通道名称，也可导出到 CSV 文件。

点击页面左上角的Import/Export，打开类似 Showfiles 和 Presets 页面的文件管理窗口。

Copy、Move、Delete、Rename 和 Read Only 功能的使用方式与 Showfiles 和 Presets 页面相同。

导入

要导入通道名称，必须将通道列表电子表格保存为 CSV 文件。主流电子表格应用均可另存为 .CSV。将 CSV 保存到 USB 驱动器并接入控制台或运行 SOLSA 的 PC；导入前无需复制到控制台内部硬盘。

在文件列表中选择 CSV 文件。下方会显示其中行列的可视化预览。使用绿色指示器标记要导入的行和列。

然后选择Import Mode。选择Add to End会将新通道添加到现有通道列表末尾；选择Overwrite会删除全部现有通道名称并替换为新名称。其他通道设置保持不变，Overwrite仅替换名称。

然后点击Import。

关闭页面左上角的Import/Export，返回Console Configuration页面。新输入通道名称会列在 Channel 列表中。

按住APPLY应用更改。

格式化通道

导入工具可自动将输入通道格式化为 Stereo、LCR、4.0 和 5.1。

例如，如果控制台将KEYS L和KEYS R识别为两个相邻通道，就会创建一个立体声KEYS通道。

要自动格式化通道，请先启用 Format Channels，再点击 Import。

对于 Stereo，相邻两个通道名称末尾必须分别为大写L和R，其余文字必须相同。导入机制也能识别 L 或 R 前的连字符（-）或下划线（_）。

LCR、4.0 和 5.1 也适用，结尾须按以下正确顺序排列。

  • LCR: L, C, R
  • 4.0: L, R, LS, RS
  • 5.1: L, C, R, LS, RS, LFE

导出

要导出通道名称列表，先选择目标驱动器，然后点击 Export 并命名文件。

控制台以 CSV 文件导出通道名称列表，其中包含通道 ID、通道名称以及 Input A 和 B 路由。

兼容模式

演出文件在整个 SSL Live 控制台系列内交叉兼容，无需额外软件或转换。

在大型控制台上创建的演出文件可加载到较小型号。如果资源不足，系统会自动进入Compatibility Mode。

控制台会尽可能创建演出文件中的路径；超出型号容量的剩余路径将被禁用。禁用路径不传递音频，但仍保存在演出文件中，并可调整推子和路径处理。Channel View 中显示如下：

也可通过Compatibility Mode中的System Options按钮手动启用兼容模式。可在处理能力较小的控制台上创建最终用于大型控制台的演出文件。状态栏出现图标表示Compatibility Mode已启用，但当前没有处理资源被禁用。

要创建超过当前型号容量的路径，必须先选择一些现有路径进行禁用。进入Console Configuration，启用Edit并点击右侧的Compatibility按钮。表格中的 Edit 列会变为Enabled。展开所需路径类型，点击各路径Enabled列中的按钮。绿色表示启用，灰色空按钮表示禁用。按住Apply确认。现在可创建更多已启用路径。

要禁用效果，请进入 Effect Rack 页面（Menu > Effects），选择要禁用的效果模块并按住其Enabled按钮。

禁用路径或效果后，Compatibility Mode按钮会变红，状态栏显示图标。Compatibility Mode允许创建相当于 L550 控制台容量的路径和效果。

只有在全部禁用路径和 FX 均已重新启用或删除后，才能关闭 Compatibility Mode。完成上述操作后，请在系统选项中关闭 Compatibility Mode 按钮。

演出文件中的禁用路径或效果下次在处理能力足够的控制台型号上加载时，会自动重新启用。
