章节：混音 | 来源： https://livehelp.solidstatelogic.com/Help/AutoFilters.html

简介

系统提供多层过滤，用于控制哪些内容会存储到自动化场景，以及从场景调用哪些内容。

将数据存储到场景时，参数更新会先经过一组细化的 Global 和 Per-Scene Store Filters （见 下文）。指定参数必须在 Global 和 Per-Scene 两种过滤器中都处于 启用 状态，才能存储到当前场景。

从场景调用数据时，先前存储的参数会经过一组类似但相互独立的 Global 和 Per-Scene Recall Filters ；这些过滤器必须处于 启用 状态，才会调用数据。

对于信号路径及其处理部分， Recall Safe 功能提供额外一层保护。必须 禁用 Recall Safe 才能调用参数。

自动化调用保护

信号路径全局调用保护

调用过滤的最高一层是 Recall Safe，它会将一条信号路径完全排除在自动化调用之外。可使用 Path Global Recall Safe逐条应用到整条信号路径。

打开 Detail View Config/Eyeconix 页面。按住 Path Global Recall Safe 即可启用。

提示： 使用 Path Link 可同时在多条信号路径上启用 Recall Safe。

Channel View 顶部的信号路径 ID 会以 紫色 突出显示，表示该路径已启用 Global Recall Safe 。该路径在 Overview 页面中也会以紫色突出显示。

Channel View： Overview：

Path Global Recall Safe 也可作为 Quick Controls 上的硬件按钮使用。通过 Quick Control 滚动按钮滚动到选项底部，或者在关闭 Follow Detail 的情况下，将 Detail View 打开到 Config/Eyeconix 区域。 Path Global Recall Safe 显示在最上方按钮上，需要按住才能触发。

注意：
Recall Safe 状态可选择存储在演出文件中。可在 Automation 页面的 Options 区域启用（MENU > Automation > Options）。

Path Global Recall Safe 会覆盖下文所述的 Global Recall Filter。这意味着，下文所有 Recall 范围参数设置会预先保留，供 Path Global Recall Safe 解除时使用。

启用 Path Global Recall Safe 只会阻止信号路径调用数据。如果 Store Filter 中已启用参数，对该路径所做的更改仍会被存储（见下文）。

信号路径分区调用保护

Recall Safe 也可应用于信号路径的单独分区，例如只保护 Fader 和 EQ，而让其他分区仍通过下述更细化的自动化过滤器进行调用。

可在以下信号路径分区启用 Recall Safe：

  • Input（Gain、Trim 等）
  • Fader 与 Mute
  • EQ 与 Filters
  • Compressor 与 Gate
  • Pan
  • Delay 与 All Pass
  • Aux 母线交叉点*

*此功能每次只对一个母线发送设置调用保护。如需对每条路径的全部 Aux 发送设置调用保护，请参阅下方 自动化过滤器 章节。

如需对某个分区启用 Recall Safe，必须将 Channel View 页面置于 Edit Safe 模式。此功能必须分配给 User Key 才能启用，例如 Channel View 中的 User Keys 弹出面板。

启用 Channel View Edit Safe Mode User Key。Channel View 中的 Input 区域（路径条顶部）、Fader、EQ/Filters、Compressor/Gate、Pan 和 Delay/All Pass 按钮会变为紫色，表示处于 Edit Safe 模式。

按下这些按钮可在所需处理分区启用 Safe 模式（可使用 Path Link 同时修改多条路径）。当某分区启用 Safe 模式时，Channel View 和 Detail View 的分区按钮会显示紫色三角形，表示调用自动化场景时该分区参数 不会 改变状态。

在 Channel View Edit Safe 模式下，Detail View 和 Quick Controls 仍会正常工作。双击路径条中部附近的情境区域，或路径条底部的 Eyeconix 区域，可打开 Detail View，在 Edit Safe 模式下更改处理设置。

完成 Safe 状态编辑后，请关闭 Channel View Edit Safe Mode User Key。

请注意，Global 和 Section Recall Safe 状态可 选择 存储在演出文件中。可在 Automation 页面的 Options 区域启用（MENU > Automation > Options）。如果不存储，Recall Safe 状态会恢复为关闭（即调用自动化场景时，参数可能根据下文所述的其他自动化过滤器设置而改变）。这样无需更改主要自动化过滤器，即可为某场演出快速、临时地阻止参数/路径发生变化。

自动化过滤器

Automation 页面（MENU > Automation）提供更细化的存储和调用过滤。

存储和/或调用场景时可排除指定功能。Store 与 Recall 可使用不同的功能过滤器：例如，希望存储某项设置但暂时不调用时，可在 Recall 页面中将其过滤掉，同时在 Store 页面中保持“未过滤”。过滤器既可全局设置，也可逐场景设置。

重要说明： 如果同一个舞台接口箱/本地模拟输入被路由到多个 Input Channel 和/或 Talkback Channel，则每个通道都可能控制与该模拟输入关联的同一组参数（增益、衰减等），因此这些通道会彼此保持同步。可在“Input”过滤器区域按通道自动化控制这些参数；但如果相关通道没有一起加入或移出 Store 和 Recall 范围，可能会无意间使共享模拟输入参数的自动化状态出现差异。触发包含这类差异的场景，可能导致模拟输入参数出现非预期行为。

将模拟输入路由到通道时初始化所生成的数据是每通道独立的，并非按模拟输入共享。因此必须确保为每个通道明确存储这些数据。

如需访问特定场景的过滤器，请触摸场景列表中的场景，再触摸 Automation 屏幕顶部的 Store Filter 或 Recall Filter 按钮，然后选择屏幕顶部的 Scene Mode 。

如需访问全局过滤器，请在相应过滤器页面中触摸 Global Mode 按钮。

随后会显示过滤器网格：

注意： 屏幕指示灯的颜色如下：Recall 过滤器为紫色（如上图），Store 过滤器为蓝色。

参数必须在全局和逐场景过滤器页面中都设为存储或调用，才能真正存储/调用。更改过滤器前必须先启用 Edit 模式（见下文）。

信号路径类型（Channels、Auxes、Masters 等）在网格中按行列出，单独元素（EQ、Fader、Mute 等）按列列出。

选择所需行并触摸屏幕顶部的 Show Details 按钮，可展开每个路径类型行，显示该类型的每条单独路径。

每个彩色指示灯表示指定信号路径或路径类型中的指定元素。灰色网格单元表示该元素不存在或已禁用。

指示灯“亮起”表示元素已包含在相应存储或调用功能中；“熄灭”表示元素已被过滤掉，即排除在存储或调用之外。

Store 过滤器

全局过滤器页面 场景过滤器页面
Global（已启用 Store*） Scene 和 Global 过滤器开启（已启用 Store）
Global 过滤器关闭（已禁用 Store） Scene 过滤器关闭、Global 过滤器开启（已禁用 Store）
不适用 Scene 和 Global 过滤器关闭（已禁用 Store）
不适用 Scene 过滤器开启、Global 过滤器关闭（已禁用 Store）
部分 Global 过滤器元素开启（这些元素已启用 Store* ） 部分 Scene 过滤器元素开启、Global 过滤器开启（这些元素已启用 Store ）
不适用 部分 Scene 过滤器元素关闭、Global 过滤器关闭（已禁用 Store）

Recall 过滤器

全局过滤器页面 场景过滤器页面
Global（已启用 Recall*） Scene 和 Global 过滤器开启（已启用 Recall）
Global 过滤器关闭（已禁用 Recall） Scene 过滤器关闭、Global 过滤器开启（已禁用 Recall）
不适用 Scene 和 Global 过滤器关闭（已禁用 Recall）
不适用 Scene 过滤器开启、Global 过滤器关闭（已禁用 Recall）
部分 Global 过滤器元素开启（这些元素已启用 Recall* ） 部分 Scene 过滤器元素开启、Global 过滤器开启（这些元素已启用 Recall ）
不适用 部分 Scene 过滤器元素关闭、Global 过滤器关闭（已禁用 Recall）

*数据必须在 Global 和逐场景过滤器页面中都设为存储/调用，因此此状态不一定表示数据实际会存储/调用；逐场景过滤器也必须启用。此外，还必须关闭相应信号路径/分区的 Recall Safe。

点击屏幕右上角的 ? 按钮可在控制台上查看这些定义。

提示：
  • 建议保持 Global 过滤器启用，以便之后作为总体 Store 和 Recall 控制使用。例如，如需在所有场景中将某通道完全且永久地排除在调用之外，只需关闭其 Global 过滤器设置。
  • 所有新场景都会使用当前场景的过滤器设置创建。
  • 使用逐场景过滤器和 All Scenes 编辑按钮（见下文），可轻松将过滤器设置传递到所有场景；之后仍可按需单独修改其中某些场景。
  • 对于无需自动化的项目，请禁用 Global Store 和 Recall 过滤器，这样只有发生预期的自动化更改（例如混音更改）时， STORE 按钮才会变为紫色。可排除的项目包括 Solo Buses、Solo Channels、Talkback Channels、Rack、Console、Fader Tile 和 Surface 设置。排除在范围之外的参数仍会存储在演出文件中。
重要说明： 调整过滤器可能导致数据丢失，使用时务必格外谨慎。
如果参数没有已存储值（即在 Store 过滤器页面中被过滤掉），但在 Recall 过滤器页面中设为调用，则会调用该参数的 默认 设置。这可能造成音频电平大幅升高，例如推子被设为 0 dB。

如需更改过滤器设置，请先启用编辑模式；触摸 Edit 按钮（屏幕右上角附近）即可启用。

通过更改编辑范围模式，可将逐场景过滤器更改仅应用于所选场景、同一 Group 中的所有场景（参阅 场景组）或列表中的全部场景；按住 Edit 按钮正上方的按钮，然后选择 Edit Single （默认）、 Edit Group 或 Edit All。之后的过滤器更改会应用到所定义的场景范围。

触摸网格中的单独指示灯可切换其状态。

也可使用每行和每列末端的圆形 Edit 指示灯更改整行和整列：

触摸任意圆形 Edit 指示灯进行选择。指示灯变红表示相应行或列正在编辑。

可选择多行和多列。触摸 Edit 标签本身还可选择全部行和列。

此时 Edit 按钮下方的三项编辑功能可用：

触摸 Set All ，将所有选中的指示项都包含在存储或调用中。

触摸 Toggle ，反转所有选中指示项的状态。

触摸 Reset All ，将所有选中指示项排除在存储或调用之外。

再次触摸 Edit 可禁用编辑模式；触摸屏幕顶部的 Scene View 按钮可返回场景列表。

注意： 上述编辑功能会应用到所有红色 Edit 指示灯已启用的行，包括位于当前可见路径类型范围上方或下方的行。
