跳线管理器

通过 Patch Manager 页面（位于 MENU > Setup > File Manager > Patch Manager 页签）可用于加载和保存当前演出的输入跳线配置。

保存输入跳线

默认文件位置为控制台内部驱动器。可使用驱动器和文件夹按钮（具体说明参见 演出文件浏览器）选择其他保存位置。要保存输入跳线，只需按下 Save As 并输入文件名。使用 Save 按钮可覆盖之前保存的跳线文件。

此操作会将当前配置的输入跳线（从舞台接口箱输入到所有通道 Input A 的路由）保存为 CSV（逗号分隔值）文件。随后可在 Microsoft Excel 等电子表格应用程序中打开该文件，以便检查或编辑。

这样可以方便地离线编辑演出文件中的输入跳线，也可将其导入具有相同通道配置的其他演出文件（有关导入跳线表文件的更多信息，请参见下文）。该文件还可作为创建更完整跳线表的起点，进一步加入话筒、话筒架等信息。

导出的跳线信息按以下字段组织：

  • Ch # ——在 控制台配置 页面中定义的通道编号
  • Ch Leg ——已格式化通道的分支编号。单声道为空；立体声通道的第一个（左）分支为 1，第二个分支为 2，以此类推。
  • Ch Name ——用户定义的通道名称；多通道时会附加分支标签（L、R 等）
  • Input A Device ——路由到此通道 Input A 所使用的 I/O 设备（舞台接口箱）
  • Input A Route ——I/O 设备上的输入编号（如果是混合信号舞台接口箱上的 AES/EBU 输入，则前缀为 AES ）
  • Device Type ——路由所连接的设备类型： Dante, MADI 或 Local （不导入）

典型文件如下所示：

[Information]
文件头信息（请勿编辑）
[PatchData]
Ch # Ch Leg Ch Name Input A Device Input A Route Device Type
1 Kick IO 1-32 1 MADI
2 Snare IO 1-32 2 MADI
3 Hats IO 1-32 3 MADI
4 Bass DI Backline-IO 1 Dante
5 1 Pads L Backline-IO AES1 Dante
5 2 Pads R Backline-IO AES2 Dante
6 Talkback Local

加载输入跳线

跳线管理器页面也可通过 Load 按钮加载 CSV 输入跳线文件。

警告： 加载 CSV 跳线文件会改变通道输入路由。请务必确认跳线表与当前使用的控制台配置正确匹配。

请注意，加载的跳线以通道 ID 编号为依据。发生以下情况时，单条路由将 不会 被导入：

  • 控制台上存在该通道 ID，但文件中不存在，或反之*
  • 控制台或文件中不存在给定通道 ID 的对应通道分支
  • 路由来源不是舞台接口箱（例如本地 I/O、其他路径、FX 机架等）
  • 文件中定义的舞台接口箱名称和类型未在控制台上配置
  • 任一条路由字段为空
  • 控制台的路由容量已用尽

* 请注意，通道 ID 编号可按以下两种方式之一显示： Default 或 Auto-Renumber，具体由 控制台配置页面定义。SSL 建议在使用跳线管理器时采用 Default 编号方式。不建议在保存和加载跳线文件之间更改编号方案。

从文件中未导入路由的控制台通道，会在加载后保留其现有控制台路由。

若要明确清除通道当前的路由，必须在NONE值输入到 Input A Route 列中。 Input A Device 和 Device Type 列可以留空。

提示： 如果演出文件是按一种场景准备、而场地采用另一种配置，可使用跳线管理器在不同舞台接口箱之间转移通道输入路由，例如从 MADI 端口转移到通过 Blacklight II 连接的舞台接口箱，或反向转移。
首先加载包含旧舞台接口箱配置的演出文件，然后按上述方法保存跳线文件，并记下旧舞台接口箱名称。在新的 I/O 端口中创建与旧舞台接口箱同名的新舞台接口箱，并从 I/O 页面删除旧舞台接口箱。最后加载先前保存的跳线文件。系统会将路由重新建立到与旧设备同名的新舞台接口箱。
请注意，此方法不会重新建立 Channel Input A 之外的输入路由，也不会重新建立通往舞台接口箱输出的路由。
