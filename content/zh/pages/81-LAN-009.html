LIVE 应用笔记：009
主题：配置 Live 控制台以使用 Dante I/O
版本：1.2

简介

本文档说明如何首次配置您的 SSL Live 控制台以接入 Dante 音频网络。

在开始配置控制台之前，重要的是要了解您的 SSL Live 控制台包含五个独立的网络设备，必须正确配置这些设备才能与外部 Dante 网络正常配合。这些设备包括：

  1. Dante Control 主网

  2. Dante Audio 主网

  3. Dante Control 备网

  4. Dante Audio 备网

  5. 连接控制网络

连接控制网络通过控制台背面三个标有 NETWORK 的端口之一访问，虽然 Dante 操作不需要该网络，但务必注意其配置，因为如果设置不正确，可能会与您的 Dante 网络发生冲突。本文档稍后将对此进行更详细的讨论。

步骤 1：设置 Dante Control 网络 IP 地址

在控制台上导航至 Menu > Options，然后选择 NETWORK 选项卡。

屏幕左侧将显示连接控制网络的信息，右侧显示 Dante Control 主网和备网的信息。

请留意：这些地址是控制台用于向 Dante 网络发送控制数据的网络适配器地址，而非 Dante 音频连接的地址。您在 DANTE CONTROLLER 应用程序中看不到这些 IP 地址。

在本示例中，我们将为 Dante Control 主网设置以下参数：IP 地址：10.101.10.21

网络掩码：255.255.0.0

网关：0.0.0.0

首先将端口类型设置为 Fixed，然后双击 IP Address 和 Netmask 框并输入上述数值。完成后，长按 Apply 按钮并等待网络适配器重新启动。

接下来为 Dante Control 备网设置以下参数：

IP 地址：10.102.10.22

网络掩码：255.255.0.0

网关：0.0.0.0

在本示例中，连接控制网络的地址设置应保持为 DHCP。如果之后需要将其设置为固定 IP 地址，请确保使用与 Dante 网络不同的子网。

连接控制网络用于连接运行 SSL SOLSA 应用程序的 PC 或平板电脑。连接到控制台的 PC 需要将其 IP 地址设置为同一子网内，才能与控制台通信。

步骤 2：设置 Dante Audio 网络和 Dante Control 网络名称

为了便于在 Dante 网络上识别控制台，控制台使用两个唯一的用户自定义名称：

  • CONTROL ID - 标识 Dante Control 网络

  • DANTE DEVICE NAME - 标识 Dante Audio 网络。这两个名称绝不可相同，这一点非常重要。

Control ID 名称可以在 NETWORK 页面中查看。以下示例中的 Control ID 为 LIVE-4C9E。

Dante Audio 网络名称可以在 I/O 页面中设置，路径为 Menu > Setup > IO，选择 Dante Expander 图标，然后双击 Device name 槽位并使用屏幕键盘输入名称。Dante (Brooklyn) 模块的名称需在 Dante Controller 中设置。

以下示例中的 Dante Device Name 为 SSL-L300-1。

在本示例中，请确保 Virtual Tie Lines 的值设置为 0。

步骤 3：设置 Dante Audio 网络 IP 地址

要设置控制台 Dante Audio 网络设备的 IP 地址，您需要一台运行 Audinate 公司 Dante Controller 应用程序的计算机，可从以下地址下载：

https://www.audinate.com/products/software/dante-controller

将计算机的网络端口设置为固定 IP 地址 10.101.10.100，子网掩码为 255.255.0.0。然后将计算机的网络端口直接连接到控制台上的 Dante Pri 端口，并启动 Dante Controller 应用程序。

在 Dante Controller 中找到控制台，本示例中为 SSL-L300-1（首次设置时此名称会有所不同），双击控制台名称打开 Device View 选项卡。

在 Device View 中选择 Device Config 选项卡。

确认 Rename Device 字段中显示的名称与 Dante Device Name 完全一致。如果名称不同，请输入正确的 Dante Device Name 并单击 Apply。在本示例中可以看到名称为 SSL-L300-1。

在本示例中，确保 Dante 设备的 Sample Rate 设置为 96kHz，如上图所示。

接下来选择 Network Config 选项卡。此页面显示控制台 Dante Audio 网络设备的 IP 地址。

在本示例中，我们将为 Primary 端口使用以下 IP 地址设置：IP 地址：10.101.10.9

网络掩码：255.255.0.0 Secondary 端口使用：

IP 地址：10.102.10.9

网络掩码：255.255.0.0

按下 Apply 使更改生效。

步骤 4：连接 SSL Network I/O 舞台接口箱（及其他 Dante 设备）

完成控制台的 Dante 设置后，即可开始向网络添加网络交换机和其他 Dante 设备。

请留意：Dante 主网和备网在物理上必须始终保持分离。

所有 SSL 舞台接口箱出厂时配置为 DHCP。要为舞台接口箱设置固定 IP 地址，请将计算机（配置为使用 DHCP 地址）直接连接到舞台接口箱，然后使用 Dante Controller 设置固定 IP 地址。更改 IP 地址后需重新为舞台接口箱上电。

因此，在本示例中，连接到 Dante 主网的任何设备必须具有以下格式的 IP 地址：

IP 地址：10.101.10.x 网络掩码：255.255.0.0

请留意：我们已在主网上使用了 IP 地址 10.101.10.9、10.101.10.21 和 10.101.10.100。

对于 Dante 备网，任何连接的设备必须具有以下格式的 IP 地址：IP 地址：10.102.10.x

网络掩码：255.255.0.0

请留意：我们已在备网上使用了 IP 地址 10.102.10.9 和 10.102.10.22。

使用 Dante Controller 将所有连接的舞台接口箱设置为 96kHz 采样率。

步骤 5：将外部 Dante 设备添加到路由菜单

Dante Expander 在 48kHz 下最多可承载 64x64 通道音频，在 96kHz 下最多可承载 32x32 通道音频。

当 Network I/O 舞台接口箱连接到网络后，必须在 I/O

页面中进行配置。按下右侧的 Dante Configuration 图标。

连接到网络的所有 Dante 设备都显示在配置页面左侧面板中。要配置设备，请按下显示屏右侧的 Edit，然后将设备从左侧面板拖放到右侧面板的 + 标志上。请留意：+ 标志会在您开始拖动设备时出现。

设备移入右侧面板后，即会出现在路由菜单中。

附录 A - 为 Live 控制台配置 DHCP Dante 网络

只有 Dante 主网可以配置为 DHCP。如果主网和备网均设置为 DHCP，控制台会自动为两个网络分配 169.254.x.x 地址，这会使主、备子网处于同一地址范围内，这种情形必须避免。

请留意：如果配置 Connectivity 端口，请使用固定 IP 地址。

设置控制台的 IP 地址

将主网设置为 DHCP，地址范围为 169.254.x.x，子网掩码为 255.255.0.0。备网地址则设置为 172.31.x.x 范围内的任意固定地址，子网掩码为 255.255.0.0。做出任何更改后，长按Apply按钮。

按此方式配置后，连接到网络的任何 DHCP 模式设备都能在主网和备网的各自正确范围内获取地址并连接到控制台。

设置 Dante Audio 网络 IP 地址

Dante Audio 网络的 IP 地址需要在 Dante Controller 中配置为 DHCP。

出厂时 Dante Audio 网络使用以下固定 IP 地址范围：

Primary 为 10.101.x.x，Secondary 为 10.102.x.x。为计算机设置固定 IP 地址并将其连接到控制台的 Dante Primary 端口。

打开 Dante Controller，双击设备以打开配置窗口，导航到 Network 选项卡。按下图所示设置 IP 地址。

单击 Reboot 完成设置。

对控制台进行电源循环后检查是否正常工作。
