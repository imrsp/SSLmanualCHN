外部控制

SSL Live 控制台可通过 Connectivity 网络端口远程控制其他设备，从而使大型系统中的不同部分实现更紧密的集成，例如控制台下游的扬声器处理单元。

下文提供受支持设备的配置说明。

控制台 OSC 路径

根据受控设备的不同，可通过控制台音频路径或专用 OSC（Open Sound Control，开放声音控制）路径控制设备。OSC 路径不具备音频处理能力，但可像其他路径类型一样放置在控制台操作界面上，便于将外部设备控制整合到工作流程中。

Console Configuration 页面（MENU > Setup > Console Configuration）中提供了按需添加 OSC 路径的区域。有关此页面的使用详情，请参阅 Console Configuration 章节。

Sigma

Sigma 设置

按照用户手册中的说明，在 Mac 或 PC 上下载并安装 Sigma Remote 应用程序 此处 .

在 Settings 页面中将协议设为 OSC；具体操作请参阅 Sigma 用户手册。

确保 Sigma 的 IP 地址适合当前网络。Sigma 默认 IP 地址为 192.168.1.201/24。

配置完成后可关闭 Sigma Remote 应用程序；通过 OSC 使用 Sigma 时无需保持该程序运行。

控制台设置

转到控制台的 外部控制 选项卡（MENU > Setup > Options）。点击 Sigma ，位于页面的 OSC SETTINGS 设备列表中。

双击 Host IP Address 字段，在下方 DEVICE CONFIGURATION 区域中输入 Sigma 的 IP 地址。Name、Incoming Port 和 Host RX Port 字段为只读，仅供参考。

输入的 IP 地址将存储在演出文件中。请注意，网络中的每个 IP 地址必须唯一。此页面也会显示控制台 IP 地址以供参考。

控制台 IP 地址必须与 Sigma 的 IP 地址位于同一子网，但不能相同；可在 Network 选项卡（MENU > Setup > Options > Network 选项卡中更改；详情见 Network Options 。

按下 Enable Control 按钮，启用控制台对 Sigma 的控制。设备列表中的 Enabled 指示灯将变为绿色。可同时启用多个设备。

创建控制台 OSC 路径

在 Console Configuration 页面（MENU > Setup > Console Configuration）中，为每个要控制的 Sigma 通道创建一条 OSC 路径。详情请参阅 Console Configuration 章节。

路径分配

使用 推子层管理器。这些路径可像其他路径类型一样按需放入推子组。可根据工作流程创建专用推子组，也可将 OSC 路径与其他控制台路径交错放置。

转到 Channel View 中的 OSC 路径，双击屏幕底部的 Eyeconix 区域以打开 Name/Eyeconix Detail View 页面。根据需要为 OSC 路径命名并设置颜色。

转到 External Control 详细视图页面，并从 Select Device 菜单中选择 Sigma。

与 Sigma 通信时，显示的 OSC 路径编号将作为要控制的 Sigma 通道编号。 Control Path Num Offset 字段可在 External Control 详细视图页面中为路径编号应用偏移。例如，若控制台 OSC 路径 8 要控制 Sigma 通道 4，则必须输入 -4 的偏移。 Path Link 可用于同时为多条 OSC 路径分配 Sigma 并应用偏移。每条路径所选设备和偏移将存储在演出文件中。

注意： 设置 Control Path Num Offset 字段时请小心，避免通过多条控制台路径访问同一个 Sigma 通道。

OSC 路径中的参数可像其他路径类型一样进行自动化；使用 Automation Filters 调整存储与调用范围。

可用控件

通过推子区块硬件及 GUI 的各推子区域（详细视图、通道视图等），控制路径的 Fader、Solo 和 Mute 控件分别控制所映射 Sigma 通道的 Gain、Solo 和 Cut 功能。

OSC 路径也可通过 SOLSA 和 TaCo 远程控制。

L-ISA

简介

SSL 与 L-Acoustics 紧密合作，使用户可从控制台界面对 L-ISA 沉浸式扬声器系统进行高度集成的控制。L-ISA 源可与单声道或立体声 Channel/Stem 关联，并可从控制台完整控制每路源的声像、距离、宽度、高度和 Aux 发送电平。

注意：此功能要求控制台软件版本为 4.11 或更高版本，L-ISA Controller 软件版本为 2.2.2 或更高版本。

L-ISA Controller 设置

打开 L-ISA Controller 软件并转到 DeskLink Settings 页面（Settings > DeskLink 选项卡）。确保 Parameters Synchronisation Mode 设置为 DeskLink OFF.

从设置选项卡顶部的 Desk Type 下拉菜单中选择 SSL Live。此前选择的控制台类型会存储在运行 L-ISA Controller 的 PC 上，并在软件启动时自动选中（默认设置为 None）。

将 Desk IP Address 输入指定字段。该地址可在控制台 Network 选项卡（MENU > Setup > Network）中查看和设置。请注意，网络中的每个 IP 地址必须唯一。

选择一个与 SSL Live 控制台通信的端口号，建议使用 8811。双击 Desk receives on port 字段并输入端口号。

控制台设置

转到控制台的 External Control 选项卡（MENU > Setup > Options）。点击 L-ISA ，位于页面的 OSC SETTINGS 设备列表中。

在 Host IP Address 字段中输入运行 L-ISA Controller 软件的计算机 IP 地址，该字段位于 DEVICE CONFIGURATION 区域。

在 Incoming Port 字段中输入与上述 L-ISA 设置相同的端口号。Name 和 Host RX Port 字段为只读，仅供参考。

输入的 IP 地址将存储在演出文件中。请注意，网络中的每个 IP 地址必须唯一。此页面也会显示控制台 IP 地址以供参考。

控制台 IP 地址必须与 L-ISA Controller 计算机位于同一子网，但不能相同；可在 Network 选项卡（MENU > Setup > Options > Network 选项卡中更改；详情见 Network Options 。

按下 Enable Control 按钮，启用控制台对 L-ISA 的控制。设备列表中的 Enabled 指示灯将变为绿色。可同时启用多个设备。

路径分配

可在控制台任意单声道或立体声 Channel/Stem 上启用 L-ISA 控制（干通道和干 Stem 除外）。

转到 External Control Detail View 页面， 从 Select Device 菜单中选择 L-ISA。 启用 DeskLink，使 L-ISA Controller 参数与所选控制台路径同步。

请注意，此时 L-ISA Controller 与控制台之间尚未建立连接（说明见下文），因此两个系统之间暂不会同步参数更改。

Path Link 可用于同时为多条路径分配并启用 L-ISA 控制。所选 External Control 设备及 DeskLink 启用状态将存储在演出文件中。

注意事项：
为使控制台 Channel 编号正确报告给 L-ISA，控制台必须设置为 使用 Console Configuration 页面中的 Default Channel 编号方式。
L-ISA 支持单声道和立体声格式的 Source。控制台路径格式必须与所关联的 L-ISA Source 匹配。
L-ISA Controller 仅通过 DeskLink 与控制台 Channel 1-256 和 Stem 1-64 通信。可在 Console Configuration 页面中将控制台路径移至编号较低的 Slot。

音频路由

控制台必须通过 MADI 连接到 L-ISA 处理器。最多可使用三路 MADI 流，将最多 96 个通道的音频从控制台路由至 L-ISA。最多添加三个 Generic MADI 设备到控制台 I/O Configuration 页面（参见 MADI Configuration ），并将相应的实体 MADI 端口连接到 L-ISA 处理器。

确保控制台与 L-ISA 处理器的采样率一致。控制台支持 96 kHz（推荐）和 48 kHz。详情见 Clocking & Sync 。

将 Channel/Stem 输出路由至已配置的 MADI 端口。详情请参阅 Routing 章节。

注意：建议将 Channel/Stem 输出直接路由至控制台 MADI 端口，并将 Channel Direct Out 设置为 Post All 处理点；详情参阅 系统选项。

建立控制台与 L-ISA Controller 之间的连接

L-ISA Controller 软件在 Parameters Synchronisation Mode 区域下提供两个初始化选项：

  • L-ISA Master ：将 Source 参数从 L-ISA Controller 发送到控制台上所有已启用 L-ISA 的路径。
  • Desk Master ：将控制台上所有已启用 L-ISA 的路径参数发送到 L-ISA Controller。

同步模式取决于工作流程；若所有 Source 参数值均存储在控制台演出文件/场景中，请使用 Desk Master 将存储的设置推送到 L-ISA Controller。若 Source 位置数据存储在 L-ISA Controller Session/Snapshot 中，则使用 L-ISA Master 选项将设置推送到控制台。

在控制台启用 L-ISA Control，并在 L-ISA Controller 中以 L-ISA Master 或 Desk Master 同步模式启用 DeskLink 后，L-ISA DeskLink Settings 页面中的 Link Status 应显示 ACTIVE，Desk 状态指示灯应亮起橙色或绿色。

Desk 指示灯为绿色时，表示所有 L-ISA Source 均已成功分配到控制台 Channel 输入通道和 Stem。若指示灯为橙色，则需要进一步操作，以正确配置 L-ISA Source 与控制台路径的分配。单击 L-ISA Controller 中的通知图标可查看错误类型。

要将控制台路径与 L-ISA Source 关联，请在 L-ISA Controller 中转到 Desk Layout 选项卡 （Sources > Desk Layout）。

任何配置不匹配（例如将单声道 L-ISA Source 分配给立体声控制台 Channel）都会以橙色警告三角形标记 配置不匹配会在 L-ISA 的 Desk Layout 选项卡中以标记；单击标记可查看详情。

可用控件

在 Channel 输入通道或 Stem 上启用 L-ISA DeskLink 后，该路径的 External Control 详细视图页面会显示 L-ISA 控制界面，可控制关联 L-ISA Source 的以下参数：

  • Pan
  • Width
  • Distance
  • Elevation
  • Aux Send （注意：这与发送到控制台 Aux 母线的电平无关）
还提供二维 XY 声像界面，可通过触控控制 Pan（X）和 Distance（Y）。

注意：控制台 XY 触控板的参数范围会自动缩放至 L-ISA Controller 的 Pan（X）和 Distance（Y）范围。参数值会从笛卡尔坐标转换为 L-ISA 使用的极坐标映射。

立体声路径还包含以下功能：

  • 提供两组上述控件，分别对应立体声对中的两个 Source；可使用二维声像界面上方的 LEFT 和 RIGHT 按钮访问。
  • 使用 LINKED 按钮启用 Link 模式后，可使用左侧控件同时调整两个 Source（右侧控件会禁用）。
  • Pan Spread 编码器可增大或减小两个 Source 之间的 Pan 位置，最大达到 L-ISA Soundscape 空间边界。此控件仅在启用 Link 模式时可用。

启用 Follow Detail 模式时，可通过主屏幕下方的快捷控制旋钮控制所选路径的 L-ISA 参数。有关快捷控制的使用详情，请参阅 Detail View 章节。

Channel/Stem 的所有常规处理和母线控件仍可正常使用。

L-ISA 参数可像其他路径参数一样进行自动化；可在 自动化筛选器页面中使用 External Control 列调整存储和调用范围。

配置 L-ISA 主电平控制

在 Console Configuration 页面（MENU > Setup > Console Configuration). 请参阅 Console Configuration 章节。

使用 推子层管理器 将 OSC 路径分配到推子区块的推子层/推子组，从而放置到控制台操作面。OSC 路径也可分配到控制台主控区块的 FOCUS 和 MAIN 推子。 请参阅 Master Tile 章节。

转到以下页面中的 OSC 路径： Channel View 中的 OSC 路径，双击屏幕底部的 Eyeconix 区域以打开 Name/Eyeconix Detail View 页面。根据需要为 OSC 路径命名并设置颜色。

转到 External Control 详细视图页面，并从 Select Device 菜单中选择 L-ISA。

OSC 路径的推子将与 L-ISA Controller 的 Master 电平位置同步，随后可用于调节该电平。

与其他路径类型一样，OSC 路径的 Fader 位置可存储在控制台自动化中；但首次连接时，其值始终从 L-ISA Controller 同步，即使同步模式设为 Desk Master 也是如此。自动化推子位置时应谨慎，以免造成大幅电平变化。

Meyer Sound Spacemap Go

简介

SSL 与 Meyer Sound 合作，使用户可从控制台界面集成控制 Spacemap Go 沉浸式扬声器系统。Spacemap Source 可与控制台 Channel 输入通道或 Stem 关联，并可完整控制每个 Source 的 XY 声像、交叉淡变、扩散和轨迹走带传输控件。

注意：此功能要求 SSL Live 控制台软件版本为 5.0 或更高版本、Meyer Sound GALAXY 固件版本为 2.4.2 或更高版本、Meyer Sound Compass 版本为 4.9.2 或更高版本，且 Meyer Sound Spacemap Go 版本为 1.2.0 或更高版本。

Meyer GALAXY 与 Spacemap Go 设置

请参考以下资料配置 Meyer Sound 系统：

  • 配置 GALAXY Processor：Spacemap Go 用户指南
  • Spacemap Go 快速入门：Spacemap Go 用户指南

在 Spacemap Go 应用程序中转到 Settings 页面，并选择左侧的 External Device Connection 区域。滚动到右侧的 Console 区域，将控制台 IP 地址输入 External Device (Client) IPv4 Address 字段。该地址可在控制台 Network 选项卡（MENU > Setup > Network）中查看和设置。网络中的每个 IP 地址必须唯一。

选择与 SSL Live 控制台通信的端口号，建议使用 4001，并将其输入 External Device (Client) Receiving Port 字段。

控制台设置

转到控制台的 External Control 选项卡（MENU > Setup > Options）。点击 Spacemap Go ，位于页面的 OSC SETTINGS 设备列表中。

输入运行 Meyer Compass 软件的计算机 IP 地址，字段为 Host IP Address 字段中输入运行 L-ISA Controller 软件的计算机 IP 地址，该字段位于 DEVICE CONFIGURATION 区域。

将上述 Spacemap Go 设置中选择的端口号输入 Incoming Port 字段。Name 和 Host RX Port 字段为只读，仅供参考。

输入的 IP 地址将存储在演出文件中。请注意，网络中的每个 IP 地址必须唯一。此页面也会显示控制台 IP 地址以供参考。

控制台 IP 地址必须与运行 Compass 和 Spacemap Go 软件的设备位于同一子网，但不能相同；可在 Network 选项卡（MENU > Setup > Options > Network 选项卡中更改；详情见 Network Options 。

按下 Enable Control 按钮，启用控制台对 Spacemap Go 的控制。设备列表中的 Enabled 指示灯将变为绿色。可同时启用多个设备。

路径分配

可在控制台任意 Channel 或 Stem 路径上启用 Spacemap Go Channel 控制（Dry Channel 和 Dry Stem 除外）。

转到 External Control Detail View 页面， 并从 Select Device 菜单中选择 Spacemap Go。

Path Link 可用于同时为多条路径分配并启用 Spacemap Go 控制。所选 External Control 设备将存储在 Showfile 中。

音频路由

确保控制台与 Meyer GALAXY Processor 的采样率一致。控制台支持 96 kHz（推荐）和 48 kHz。详情见 Clocking & Sync 。

Meyer GALAXY Processor 支持模拟、AES3 和 AVB 输入。将控制台 Channel/Stem 输出路由至连接 Meyer Processor 所需的端口。详情请参阅 Routing 章节。

注意：建议使用 Channel/Stem Direct Output 路由至 Meyer 系统，并设置为 Post All 处理点；详情参阅系统选项。

可用控件

在 Channel 或 Stem 上启用 Meyer Sound Spacemap Go 后，该路径的 External Control 详细视图页面会显示 Spacemap Go 控制界面，可控制关联 Spacemap Go 通道的以下参数：

  • X Y Pan
  • Crossfade
  • Spread
  • Trajectory Rate
  • Trajectory 传输控件
提供二维声像界面，可通过触控控制 Spacemap 中的 X、Y 位置。

请注意，Spacemap Go iPad 应用程序具有控制优先权；在应用程序中调节参数时，控制台控件会暂时禁用。

Channel/Stem 的所有常规处理和母线控件仍可正常使用。

Spacemap Go 参数可像其他路径参数一样进行自动化；可在 自动化筛选器页面中使用 External Control 列调整存储和调用范围。

但是，SSL 和 Meyer Sound 建议在控制台自动化中将 External Control 参数设为 disabled ；对使用 Spacemap Go 的路径，应改在 Meyer Sound 自动化系统中自动化 Spacemap Go 参数。

配置 Spacemap Go 主电平控制

在 Console Configuration 页面中创建一条用于 Spacemap Go Mix Master 电平控制的 OSC 路径（MENU > Setup > Console Configuration). 请参阅 Console Configuration 章节。

使用 推子层管理器 将 OSC 路径分配到推子区块的推子层/推子库，从而放置到控制台操作面。OSC 路径也可分配到控制台主控区块的 FOCUS 和 MAIN 推子。 请参阅 Master Tile 章节。

在 Channel View 中转到该 OSC 路径，双击屏幕底部的 Eyeconix 区域，打开 Name/Eyeconix Detail View 页面。根据需要为 OSC 路径命名并设置颜色（例如 SMG Master）。

转到 External Control 详细视图页面，并从 Select Device 菜单中选择 Spacemap Go。

OSC 路径的推子位置将与 Spacemap Go Mix Master 电平位置同步，随后可用于调节该电平。

与其他路径类型一样，OSC 路径的推子位置可存储在控制台自动化中；但与上述逐路径 Spacemap Go 控件相同，建议将 External Control 参数设为 disabled 。自动化推子位置时应谨慎，以免造成大幅电平变化。

自动化 Spacemap Go 参数

为避免数据冲突，Spacemap Go 参数应仅存储在一个自动化系统中：Spacemap Go（推荐）或控制台。

要禁止从控制台自动化系统调用，请转到 Automation 页面（MENU > Automation) 并选择 Global Recall Filters 页面（参阅自动化筛选器）。找到已启用 Spacemap Go 控制的路径，并确保这些路径的 External Control 列处于关闭状态。

注意：以上操作只会禁止调用 External Control 参数。要同时禁止存储这些参数，还需在 Global Store Filters 页面中关闭 External Control 列。

使用 Generic OSC 路径触发 Spacemap Go Snapshot

要在触发控制台场景时同步触发 Spacemap Go Snapshot，可使用分配给 OSC 路径的 Generic OSC 设备（设置详情见下文）。

将 Generic OSC 设备配置为与上述 Spacemap Go 设备相同的 IP 地址和端口号。

配置一个 Generic OSC 推子，并将其 OSC Address 设为 /Console/recall/system。Spacemap Go 系统的 Snapshot 可设置 900 到 3000 之间的 External Recall ID。设置推子的 Min 和 Max 字段，以覆盖所需的 Snapshot 数量。推子以百分比表示，因此 Min/Max 范围 900 至 1000 表示推子每变化 1 %，External Recall ID 从 900 起增加 1。例如推子值 42 % 对应 Snapshot #942（900+42）。

提示：Generic OSC 推子 1 分配到 OSC 路径时会映射到实体推子，因此建议使用其他推子编号，避免误动实体推子导致 Snapshot 更改。

在 Console Configuration 页面中新建 OSC 路径（MENU > Setup > Console Configuration). 请参阅 Console Configuration 章节。

确保 OSC 路径的 External Control 参数已设置为在控制台自动化场景中存储和调用。详情参阅自动化筛选器。

使用 推子层管理器 将新 OSC 路径放置到控制台操作面，便于访问；也可使用 Layers 按钮访问。

在 Channel View 中转到该 OSC 路径，双击屏幕底部的 Eyeconix 区域，打开 Name/Eyeconix Detail View 页面。根据需要为 OSC 路径命名并设置颜色（例如 SMG Snapshot）。

转到 External Control 详细视图页面，并从 Select Device 菜单中选择 Generic OSC。

在每个需要触发 Spacemap Go Snapshot 更改的控制台场景中，将 OSC 推子位置改为所需的 Spacemap Go External Recall ID，并注意上述偏移和 Min/Max 范围。请记得 Store 将此值存储在控制台自动化系统中，以便触发场景时调用。

提示：点击 OSC 推子将其选中，再双击详细视图右下角的文本字段，调出键盘并输入指定值。详情参阅详细视图。

触发包含 Generic OSC 路径中 OSC 推子新值的场景时，会向 Spacemap Go 发送 OSC 消息，使 Snapshot 更改与控制台自动化场景更改同步。

d&b Soundscape 控制

简介

SSL 与 d&b 合作实现集成控制，用户可直接从 SSL Live 控制台在 d&b Soundscape 沉浸式扬声器系统中定位对象。每台 DS100 处理引擎最多可配置 64 个对象，均可由控制台控制。对象可由 Channel 输入通道或 Stem 组母线控制，并可在通道详细视图中完整控制 XY 声像、扩声、En-Space 发送、延迟模式和映射区。

“d&b Soundscape 由 DS100 Signal Engine 驱动，该革命性音频系统处理器基于强大的 Dante 信号矩阵，并通过 En-Scene 和 En-Space 两个软件模块提供卓越的创作能力。”

d&b DS100 Processor 内置 Dante I/O（48 kHz 和 96 kHz 均为 64 个通道）。建议通过 SSL Live 控制台的 Dante 选件连接音频。控制通信通过控制台 Connectivity 网络直接与 DS100 建立。

注意：此功能要求 SSL Live 控制台软件版本为 5.2 或更高版本，

d&b Soundscape 设置

在控制台 Setup 菜单中转到 Options > External Control。从 OSC Settings 列表选择 d&b Soundscape，以在 DEVICE CONFIGURATION 面板显示详情。

  • Enable Control：开启或关闭 d&b Soundscape 的 OSC 通信。
  • Host IP Address - 需要输入 DS100 设备的 IP 地址。（默认 50010） 
  • Incoming Port - 直接连接 DS100 时不需要；配合 d&b 软件桥接应用程序 Remote Protocol Bridge 使用。 
  • Host RX Port - 直接通信时需设置为 DS100 的入站端口。 

输入的 Host IP 地址将存储在演出文件中。网络中的每个 IP 地址必须唯一。此页面也会显示控制台 IP 地址。控制台与 DS100 必须位于同一子网，但 IP 地址不能相同；可在 Options > Network 选项卡中修改控制台地址。

音频路径映射

可在任意控制台 Channel 或 Stem（Dry 类型除外）上启用 Soundscape 控制。

转到 External Control 详细视图页面，并从 Select Device 菜单中选择 d&b Soundscape。Control Path Num Offset 字段可在向 OSC 设备发送路径编号时添加偏移。例如使用 48 路单声道 Channel 和 8 路 Stem 时，Channel 使用偏移 0（d&b Source ID 1-48），所有 Stem 取偏移 48（d&b Source ID 49-64）。

路径联动可用于同时为多条路径分配并启用 d&b Soundscape 控制。每条路径所选外部控制设备将存储在演出文件中。

音频路由

DS100 配有 Dante 接口；与控制台之间的音频连接可使用控制台内置 Dante Expander 或高容量 BLII/X-Light Bridge。音频可通过 Virtual Tie Line（由 Dante Controller 管理）或控制台软件创建的 Dynamic Dante Route（双域路由）连接。要从控制台进行 Dynamic Dante Routing，必须在 Dante Configuration 页面将 DS100 添加到 Configured Devices。配置后，可使用任意控制台路由界面连接 DS100 的路径输入输出；这些 Dynamic Dante 路由会随演出文件存储和调用。

d&b Soundscape 控件

在控制台路径详细视图中，d&b 控件显示在触控界面上，也可在 Follow Detail 模式下通过编码器控制。可控制以下参数：

  • X 和 Y 相对于映射区的坐标（可使用二维位置图、两个滑块，或 Follow Detail 模式下的两个旋钮控制）。
  • Spread （使用位置图上的捏合手势、专用滑块或 Follow Detail 模式下的旋钮控制）。
  • En-Space Send （使用位置图上的捏合手势、专用滑块或 Follow Detail 模式下的旋钮控制）。
  • Delay Mode （使用屏幕按钮或 Follow Detail 模式下的硬件开关，在 Off、Tight 和 Full 之间切换）。
  • Mapping Area （按住以选择 Area 1-4，详情见 d&b Soundscape 文档）。

多个通道的控件可通过路径联动关联并成组控制，方式与控制台参数相同。

Soundscape 参数可像其他路径参数一样进行场景自动化；可在 自动化筛选器页面中使用 External Control 列调整存储和调用范围。

配合 d&b Remote Protocol Bridge（RPB）的高级用法

对于由多个用户界面控制的工作流程（例如控制台和 R1），需要与控制台进行双向通信。DS100 不直接使用 OSC 通信，因此需要 d&b Remote Protocol Bridge 应用程序。

d&b Remote Protocol Bridge 还支持两台 DS100，可并行配置用于冗余，也可作为两台独立设备，在 Soundscape 系统中提供最多 128 个声音对象。

有关包含 Remote Protocol Bridge 的使用场景，请联系当地 SSL 代理。

通用 OSC

简介

控制台可通过 UDP 网络协议控制许多支持 OSC（Open Sound Control）标准的设备。

请参阅各设备手册，确认设备是否支持 OSC 远程控制并查看设备专用设置说明。

要从控制台操作界面通过 OSC 控制第三方设备，需要以下信息：

  • 受控设备的 IP 地址
  • 控制台接收端口号（即设备发送 OSC 数据包的目标端口）
  • 设备接收端口号（即控制台发送 OSC 数据包的目标端口）
  • 受控参数的 OSC 方法地址和数据类型

有关 OSC 协议的更多详情，请参阅 Open Sound Control 网站（www.opensoundcontrol.org）。

控制台设置

转到控制台的 External Control 选项卡（MENU > Setup > Options）。点击 Generic OSC ，位于页面的 OSC SETTINGS 设备列表中。

在 DEVICE CONFIGURATION 区域提供的 Host IP Address、Incoming Port 和 Host RX Port 字段中输入设备 IP 地址和端口号。

某些第三方设备要求控制台通过回送接收值来响应 OSC 消息。此时请确保 Enable Parameter Echo 按钮已启用。详情请参阅设备手册。如不确定，请先将 Enable Parameter Echo 设为关闭；仅在其他设置均正确但仍无法与设备通信时启用。

输入的设置将存储在 Showfile 中。网络中的每个 IP 地址必须唯一。此页面也会显示控制台 IP 地址以供参考。

控制台 IP 地址必须与 OSC 设备位于同一子网，但不能相同；可在 Network 选项卡（MENU > Setup > Options > Network 选项卡中更改；详情见 Network Options 。

定义 OSC 方法

每个第三方设备 Channel 最多可定义八个 Fader（连续可变）参数和八个 Switch 参数，供控制台控制。这些参数在 DEVICE CONFIGURATION 区域的相应选项卡中定义。

为每个参数输入以下信息：

  • Label ：控制台界面中控件旁显示的文本。字符 * 会替换为发起控制的控制台路径编号（包括 Path Num Offset；见下方 Path Assignment）。例如输入“Mute ch *”，路径 1 显示“Mute ch 1”，路径 2 显示“Mute ch 2”，依此类推。
  • OSC Address ：要控制的 OSC 方法地址。控制台仅处理浮点数据类型。字符 * 会替换为发起控制的控制台路径编号（包括 Path Num Offset）。例如输入“/solidstatelogic/sigma/channels/fader/*”，路径 1 与 SSL Sigma Fader 1 的 OSC 方法通信，路径 2 与 Fader 2 通信，依此类推。
  • Min 和 Max 值
    • Fader：在 Min 与 Max 之间进行线性插值。
    • Switch：Off = Min，On = Max。通常分别为 0 和 1；请参阅设备手册。
  • Colour ：控件在控制台界面中的显示颜色。

Switch 选项卡还可将数据类型设为 Integer，使 Switch 仅响应和发送带整数参数的 OSC 消息。

按住 Apply 按钮以存储参数。

Use the Clear Pending 按钮，将参数列表重置为当前已应用状态。

按下 Enable Control 按钮，启用控制台对 OSC 设备的控制。设备列表中的 Enabled 指示灯将变为绿色。可同时启用多个设备。

创建控制台 OSC 路径

Generic OSC 设备既可从控制台音频路径控制，也可从不包含控制台音频处理的专用 OSC 路径控制。选择哪种方式取决于第三方设备是否直接与控制台音频路径交互，例如插入特定通道的外部效果处理器；或是否与特定音频路径无关，例如用于触发灯光 Cue 的演出控制系统。

可同时从音频路径和 OSC 路径控制，但必须确保它们寻址第三方设备上的不同通道；详情见下方路径分配章节。

如需专用 OSC 路径，请在 Console Configuration 页面中创建（MENU > Setup > Console Configuration). 请参阅 Console Configuration 章节。

路径分配

使用以下功能将音频/OSC 路径放置到操作界面： 推子层管理器。OSC 路径可像其他路径类型一样按需放入推子组。可创建专用推子组，也可与其他控制台路径交错放置。

转到以下页面中的路径： Channel View 中的 OSC 路径，双击屏幕底部的 Eyeconix 区域以打开 Name/Eyeconix Detail View 页面。根据需要为路径命名并设置颜色。

转到 External Control 详细视图页面，并从 Select Device 菜单中选择 Generic OSC。

与第三方设备通信时，显示的路径编号将作为受控设备通道编号。 Control Path Num Offset 外界控制详细视图页面中的此字段可为路径编号应用偏移。例如，若控制台 OSC Path 8 要控制设备 Channel 4，则必须输入 -4 的偏移。 Path Link 可用于同时为多条 OSC 路径分配 Generic OSC 并应用偏移。每条路径所选设备和偏移将存储在演出文件中。

注意：请确保不会通过多条控制台路径访问同一设备通道，包括将同一设备通道分配给不同路径类型（如 OSC 路径与通道），或通过 Control Path Num Offset 字段产生重复映射。

OSC 路径中的参数可像其他路径类型一样进行自动化；使用 Automation Filters 调整存储与调用范围。

可用控件

Generic OSC 设备设置中配置的参数（最多八个推子和八个开关）可从 External Control 详细视图页面访问，适用于任何配置为控制 Generic OSC 设备的路径。

左侧图形声像器表示前三个已配置的推子参数，标记为 X、Y、Z，可图形化表示许多支持 OSC 的第三方沉浸式音频处理器的 3D/环绕声像。

可通过右下角的 Display X-Z 和 Display Y-Z 按钮选择图形声像器显示的维度。

分配给 OSC 路径后，该路径的 Fader 将控制设备中配置的第一个推子参数。该路径的 Mute 和 Solo 开关分别控制第一和第二个开关参数，可通过推子区块硬件及 GUI 的各推子区域（详细视图、通道视图等）访问。
