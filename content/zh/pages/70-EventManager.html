事件管理器

事件管理器可用于在控制台上编排由状态驱动的操作。事件可在 Event Manager 页面（Menu > Tools > Event Manager）中配置。页面上半部分显示所有已配置事件的列表。启用侧边栏中的 Edit 按钮即可修改事件列表。

按下 New Event 可创建新事件。事件在屏幕下半部分配置。双击页面中央的名称字段可重命名事件。在此添加事件触发条件（源）以及触发条件满足时要执行的操作（目标），从而定义事件功能。

要删除事件，请在侧边栏中启用 Edit 模式，并使用事件列表 Edit 列中的圆形按钮，将相应事件置于可编辑状态。长按侧边栏中的 Delete 按钮可删除所选事件。

注意：事件删除后无法撤销！

要禁用事件而不将其删除，请按上述删除操作的方法将列表中的事件置于可编辑状态，然后按下侧边栏中的 Disable 按钮。已禁用的事件会在事件列表的 Enabled 列中以红色图标标识。有关禁用单个 事件源 和 事件目标的方法，请参见下文。

要复制事件，请按上述删除操作的方法将该事件置于可编辑状态，然后按下侧边栏中的 Duplicate Event 按钮。系统会创建一个副本，并在原事件名称后附加编号，例如“New Event (1)”。

每个事件的当前状态显示在 State 列中。事件处于活动状态（即所有源条件均已满足）时，指示灯会亮绿色。

事件源

使用 Add Source 按钮添加新的事件源。每个事件最多可添加 12 个源。选择 Sources 区域右上角相应的 Any 或 All 条件，并设置 Trigger 选项，即可指定满足任一或全部源条件时触发事件。当任一或全部事件源状态变为真时，事件状态也会变为真，并触发所有事件目标操作。

使用 Remove Source 按钮从事件中删除所选源。

注意：删除源后无法撤销，必须手动重新添加。

使用 Disable 按钮可禁止所选源触发事件，而无需永久删除。这样可以暂时从事件逻辑中移除某些源（例如源产生错误信号时），同时保留其设置。

每个源的当前状态由事件列表和源详细信息区域中的图标表示。绿色图标表示源条件已满足，并会参与激活事件。

自动化输出

控制台的自动化场景可用于触发事件。请按以下步骤配置：

在 Event Manager 页面中选择 Automation Output 源类型。

现在可将此 Automation 事件源链接到场景的 Output Action。在控制台的 Automation 页面中选择所需场景，进入 Output Actions 区域并选择 Event 操作类型。

从 LINKABLE EVENTS 列表中选择事件，然后按下 Apply。

Arm 对 Output Action 执行 Arm，以确保场景触发时该操作会执行。

已链接的场景会显示在 Event Manager 页面中。

GP 输入（GPI）

GPI 源可通过控制台的本地 GPIO 端口*，或任何带 GPIO 端口的 SSL Live MADI 或 Network I/O Dante 舞台接口箱触发事件。

选择 GP Input 源类型，然后使用 Route 按钮打开 Routing 对话框，其中列出控制台可用的所有 GPI。选择要使用的 GPI 并按下 Make。关闭 Routing 对话框，返回事件源配置。

GPI 可按多种不同的 Action 方式触发：

  • Latching ：GPI 信号变为高电平时，源状态变为真并保持为真
  • Momentary ：仅在 GPI 信号为高电平期间，源状态才为真
  • Momentary-On-Hold ：GPI 信号变为高电平时源状态变为真；若 GPI 信号在 500 毫秒后变为低电平，源状态变为假，否则保持为真

使用 Inverted 按钮可反转输入事件触发器的 GPI 状态；即当传入 GPI 信号变为低电平时，源状态变为真。

MIDI 输入

MIDI 源可通过控制台本地 MIDI 端口*、任意 SSL Live MADI 舞台接口箱上的 MIDI 端口或 ipMIDI 触发事件。

选择 MIDI Input 源类型，然后使用 Route 按钮打开 Routing 对话框，其中列出控制台可用的所有 MIDI 端口。选择要使用的端口并按下 Make。关闭 Routing 对话框，返回事件源配置。

注意： 如果某个 MIDI Input 已路由至 MIDI 端口，之后添加的 MIDI Input 会自动路由至最后使用的 MIDI 端口。

Note On, Note Off, Program Change 和 Control Change （CC）消息均可选择用于事件。

源区域的其余部分用于配置预期接收的 MIDI 消息（MIDI Channel、Note Number 等，视情况而定）。

MIDI 事件源具有 Pulse Fire 选项。启用后，收到指定类型的消息时，源状态会变为真并持续 500 毫秒，然后变为假，而不受输入状态影响。对于 Program Change 消息，这允许重复输入再次触发事件。若未启用 Pulse Fire，Program Change 后事件源状态会保持高电平。

注意： 如果使用 ipMIDI 作为事件源，应使用不同于以下功能所用端口的 ipMIDI 端口： 自动化输入/输出操作, DAW 控制 and 时间码。

用户按键输入

控制台的 用户按键 也可用于触发事件。选择 User Key 源类型，然后选择触发事件所需的 Action。

可用的不同 Action 如下：

  • Latching ：按下 User Key 时，源状态变为真并保持为真
  • Momentary ：仅在按住 User Key 期间，源状态才为真
  • Momentary-On-Hold ：按下 User Key 时源状态变为真；若在 500 毫秒后释放 User Key，源状态变为假，否则保持为真

每个事件只能添加一个 User Key 源。该事件的 User Key 源随后会像通常一样出现在控制台任意 User Key 的分配选项中；可从相应的 用户选项页面 （MENU > Setup > Options > USER KEYS）进行分配。

推子源

当信号路径推子达到指定的 dB Threshold时，可使用 Fader Function 源触发事件。

推子功能需要输入具体的信号路径编号（也可启用 Any Path ，在所选类型的任一信号路径达到指定状态时触发事件源）。

选择一个或多个信号路径后，设置 Threshold ，并指定源应在高于 Trigger Above 还是低于 Trigger Below 阈值时触发。

注意： 推子事件源依据的是计入所有相关 VCA 电平 之后 的推子位置。
例如：若信号路径推子为 -6 dB，作用于该路径的 VCA 为 -10 dB，则用于判断推子位置高于还是低于事件阈值的数值为 -16 dB。

事件目标

使用 Add Destination 按钮添加新的事件目标。每个事件最多可添加 12 个目标。事件状态变为真时，会执行所有目标操作。

使用 Remove Destination 按钮从事件中删除所选目标。

注意：删除目标后无法撤销，必须手动重新添加。

使用 Disable 按钮可从事件中禁用所选目标，而无需永久删除。这样可以暂时停止触发某些目标，同时保留其设置。

使用源区域中的 Test 按钮测试事件状态变为真（以测试所有已配置目标）。使用目标区域中的 Test 按钮测试所选目标。

添加新目标时，使用目标区域左侧的 TYPE 选项选择目标类型。可用目标类型如下：

None （未分配）
Automation Input
GP Output (GPO)
MIDI Output
Console Function
Fader Function
注意： 事件目标 不得 设置为与事件源相同的功能，否则会形成反馈循环！

自动化输入

事件可用于触发指定的自动化场景。请按以下步骤配置：

在 Event Manager 页面中选择 Automation Input 目标类型。

现在可将此 Automation 事件目标链接到场景的 Input Action。在控制台的 Automation 页面中选择所需场景，进入 Input Actions 区域并选择 Event 操作类型。

从 LINKABLE EVENTS 列表中选择事件，然后按下 Apply。

Arm 对 Input Action 执行 Arm，以确保事件激活时触发场景。

已链接的场景会显示在 Event Manager 页面中。

GP 输出（GPO）

GPO 目标可通过控制台的本地 GPIO 端口*，或任何带 GPIO 端口的 SSL Live MADI 或 Network I/O Dante 舞台接口箱控制外部设备。

选择 GP Output 目标类型，然后使用 Route 按钮打开 Routing 对话框，其中列出控制台可用的所有 GPO。选择要使用的 GPO 并按下 Make。关闭 Routing 对话框，返回事件目标配置。

GPO 行为可设置为多种不同的 Action：

  • Latching ：事件状态变为真时切换 GPO 状态
  • Momentary ：GPO 状态跟随事件状态
  • Pulse ：事件状态变为真时，GPO 闭合脉冲持续 500 毫秒
  • Turn On ：事件状态变为真时 GPO 闭合；需要第二个事件才能重新断开 GPO
  • Turn Off ：事件状态变为真时 GPO 断开；需要第二个事件才能重新闭合

使用 Inverted 按钮可反转 GPO 状态与事件状态的对应关系；即事件状态变为真时 GPO 断开，反之亦然。

MIDI 输出

MIDI 目标可通过控制台本地 MIDI 端口*、任意 SSL Live MADI 舞台接口箱上的 MIDI 端口或 ipMIDI 控制外部设备。

选择 MIDI Output 目标类型，然后使用 Route 按钮打开 Routing 对话框，其中列出控制台可用的所有 MIDI 端口。选择要使用的端口并按下 Make。关闭 Routing 对话框，返回事件目标配置。

如果某个 MIDI Output 已路由至 MIDI 端口，之后添加的 MIDI Output 会自动路由至最后使用的 MIDI 端口。

Note On, Note Off, Program Change 和 Control Change （CC）消息均可选择用于事件。

目标区域的其余部分用于配置要发送的 MIDI 消息（MIDI Channel、Note Number 等，视情况而定）。

注意： 如果使用 ipMIDI 作为事件目标，应使用不同于以下功能所用端口的 ipMIDI 端口： 自动化输入/输出操作, DAW 控制 and 时间码。

  • Follow On ：功能状态跟随事件状态
  • Follow Off ：功能状态跟随事件状态的反相
  • Toggle ：每当事件状态变为真时切换功能状态

推子目标

事件可通过 Fader Function 事件目标，将信号路径推子设置到指定电平。

推子功能需要输入具体的信号路径编号（也可启用 All Paths 以影响所选类型的所有信号路径）。

选择一个或多个信号路径后，为其设置所需的 dB 值。

注意： 推子事件目标设置的是计入所有相关 VCA 电平 之前 的推子位置。
例如：若事件将某信号路径推子设为 -6 dB，而作用于该路径的 VCA 设为 -10 dB，则总推子增益为 -16 dB。


* L100 没有本地 GPIO 或 MIDI 端口；在 L100 上只能配置使用 ipMIDI 或远程舞台接口箱 GPIO 和 MIDI 的事件。
