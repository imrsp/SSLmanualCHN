路由

Live 控制台的信号路由极其灵活，几乎可以将任何信号路由到任何位置！ 例如，舞台接口箱输入可以直接路由到效果器机架模块 或通道的动态 Key Input，无需占用另一个控制台通道 （如有需要，请使用 I/O 页面设置前置放大器增益）。

可通过两种方式进行路由：

  • 通过 路由视图：为某条信号路径建立一条输入或输出路由
  • 通过 XY路由（见下文）：同时建立多条路由（例如将舞台接口箱批量路由至通道输入时）

有关 I/O 配置的更多信息，请参阅时钟与 I/O 连接章节。

路由视图

此 路由视图 会出现在控制台的许多位置。 最常见的位置包括：

  • 详细视图 Input/Routing 区域的Route Input A, Route Input B 和 Route Output 按钮
  • 详细视图 Insert A/Insert B 区域的
    • Route Send 和 Route Return 按钮
  • 路由矩阵输入和输出时
  • 为动态 FX 单元路由 Key Input 时

注意：输出可路由至任意数量的目标。除 Channel 外，所有信号路径类型的输出均位于全部处理（包括推子）之后；Channel 可按通道设置或设为默认值。

例如，要为通道主输入（Input A）选择信号源，请触摸 Route Input A；该按钮位于输入/路由详细信息对话框中。

此时将显示 Routing View：

触摸左列中的某个组，其内容会在下一列打开；如有需要，可继续在第三列和第四列中展开。

如有需要，可滑动列中的列表以向上或向下滚动。

对于舞台接口箱或通用 MADI/Dante 设备，第一列显示物理设备，第二列显示其 I/O 类型，下一列显示各个 I/O。

要使用控制台内部 I/O，请触摸控制台组（显示为 LIVE 及 ID 编号），然后选择信号源类型和编号。若选择通道或母线信号，还可继续选择通道中的可用信号源点。

按 Make 完成路由。

注意：路由多通道输入或输出（Stereo、LCR 等）时，按住 Make 按钮，再按 Make All，即可在条件允许时将信号路径的所有分支路由至连续的物理接口。
也可使用下文介绍的 XY Routing 路由至连续接口。
或者在 Main 屏幕上选择通道条，然后按右侧的 Spill 按钮，依次路由各个分支。

路由框的颜色含义如下：

绿色：设备或设备元素处于活动状态（已检测并锁定），且包含与此通道之间的路由；
青色：设备或设备子类别处于活动状态并可用于路由，但自身不是可路由元素；
蓝色：可路由的设备元素处于活动状态，可与此通道建立路由；
灰色：设备或设备元素未激活（未检测到或未锁定），且不包含与此通道之间的路由。
棕色：设备或设备元素处于活动状态，但已被占用，因此无法用于路由。
黄色：设备或设备元素未激活但已被占用。此设备上不会有音频。
红色：设备或设备元素未激活，但包含与此通道之间的路由。此设备上不会有音频。

取消路由分为三个层级：

对于 Route In：

  • Clear <element>（例如 Clear Analogue A 11）取消从所选设备元素/物理接口发出的所有路由（这些路由显示在 Clear <element> 按钮左侧可滚动的当前路由列表中），包括已连接到其他通道的路由；
  • Unmake 仅取消所选通道从当前高亮（已选择）设备元素取得的输入路由。路由到同一信号源的其他通道不受影响；
  • Clear Input A/Clear Input B 取消该通道的所有输入路由，即使它们当前未被选择。这包括立体声/LCR 通道的所有分量，但不包括该通道的另一路（A 或 B）输入。

对于 Route Out：

  • Clear <element>（例如 Clear AES 5/6）取消通往所选设备元素/物理接口的所有路由；
  • Unmake 取消该通道通往当前高亮设备元素的输出路由；
  • Clear Output 取消该通道的所有输出路由，即使它们当前未被选择。这包括立体声/LCR 通道的所有分量。

要关闭此显示，请选择其他 Detail 页面、取消选择 Route，或触摸 Close 按钮。

注意：路由列上方的 Close 按钮仅关闭路由界面，Detail 对话框仍保持打开。 触摸路由按钮上方的 Close 会关闭整个 Detail 对话框。


XY 路由

此 XY Routing 页面（MENU > Setup > XY Routing）提供网格式批量路由界面。

XY Routing 页面可概览跳线状态，并能快速完成大量、复杂的跳线分配。

首先选择所需的 Source 和 Destination，二者位于页面左上角。 此 Source 列在 y 轴上，Destination 列在 x 轴上。

轻触任意交叉点可创建一条待建立路由（青色）。按住 Apply 建立路由。 如果设备在线且已锁定，路由会变为绿色；如果设备离线或未锁定，路由会变为红色。

要同时进行多项分配，请按住第一个交叉点，再触摸最后一个交叉点，然后同时松开双指。 也可轻触第一个交叉点，再按 1:1 From Selected。界面会显示一系列 1:1 待建立路由（青色）。按住 Apply 建立这些路由。

要清除路由，请轻触交叉点；交叉点将显示为空心圆（待取消路由状态）。按住 Apply 执行更改。

也可按住 Clear All Routes From...，清除来自所选信号源（例如整台舞台接口箱）的全部路由， 或按住 Clear All Routes To...，清除通往所选目标（例如所有通道的 Input A）的全部路由。 注意：这些 Clear All... 按钮会立即取消路由，而 Apply 按钮无需按下。

当输入或输出已与某一位置建立路由时，坐标轴上的名称会变灰。如果一个输入被路由到多个位置，坐标轴上的名称会变为橙色。

如果某个输入已路由至一个或多个目标，仍可从此输入建立通往更多目标的附加路由。

如果某个输出已接收来自其他信号源的路由，在 XY Routing 中为该输出建立新路由会覆盖原有路由。 只有 XY Routing 可以覆盖路由。在普通路由视图中，必须先取消通往该输出的路由，才能建立来自其他信号源的新路由。

轻触坐标轴上的信号源/目标点，可在左上角详细信息框中显示当前路由。

页面右侧设有导航按钮，用于浏览较长的列表。 Align Selected 会移动网格，使所选交叉点位于左上角。 Home 会将网格恢复到默认位置（网格左上角）。

使用 Refresh I/O 按钮刷新 XY Routing 页面。

路由模式

XY Routing 包含两种路由模式：

  • Showfile Saved：标准模式。
  • Outside of Showfile：仅用于 Dante 的持久路由。

控制台可直接提供完整 Dante 路由。您可以选择将路由存储在演出文件内，也可以将其配置为独立的系统级 Dante 路由。控制台由此可作为功能强大的 Dante 控制器，无缝管理系统级路由。

SSL Live 控制台软件完整集成 Dante 路由，将跳线配置无缝嵌入演出文件，并通过单一用户界面实现完整路由控制。即使处于离线状态，也可直接在控制台上配置所有 Dante 设备并进行跳线分配。控制台的离线编辑器 SOLSA 同样具备此功能，可提前完成整场演出的准备工作。Dante 可作为控制台的可扩展延伸，使 SSL Live 提供完全受控制台管理的强大音频路由方案。

这一功能也带来更高灵活性：可按需手动覆盖路由和 Dante 订阅，但控制台绝不会自动覆盖设置。软件采用“先到先得”的自动仲裁机制；控制台关机或演出文件卸载时会释放 Dante 订阅，为其他控制台或其他活动信号释放资源。所有内容均存储在演出文件中，可完整调用。

需要永久信号路由时，可使用 SSL Live 的 Outside of Showfile Routing Mode。在此模式下建立的订阅，即使控制台关闭或加载其他演出文件仍会保持有效，因此无需为了持久的“场馆固定路由”而打开 Dante Controller。

故障排除

如果控制台无法建立全部或部分 Dante 路由，将出现弹窗。请查看屏幕左上角的 Status Message 区域了解详情，其中会说明路由未能建立的原因。

显示的消息将是以下之一：

消息 含义
Warning: Some routes have not been made:
Insufficient Dante dual-domain inputs.		 尝试建立的路由中，一部分已成功建立，另一部分未能建立。
原因是剩余的双域输入不足，无法建立全部路由。
Warning: Some routes have not been made:
Insufficient Dante dual-domain outputs.		 尝试建立的路由中，一部分已成功建立，另一部分未能建立。
原因是剩余的双域输出不足，无法建立全部路由。
Warning: No routes have been made:
Insufficient Dante dual-domain inputs.		 尝试建立的路由均未建立。
原因是剩余的双域输入不足，无法建立任何路由。
Warning: No routes have been made:
Insufficient Dante dual-domain outputs.		 尝试建立的路由均未建立。
原因是剩余的双域输出不足，无法建立任何路由。

双击 Status Message 可打开 Status Message 窗口，其中显示自上次启动以来的所有状态消息。

每条失败的 Dante 路由都会对应一条 Status Message。向下滚动列表可查看失败路由；未列出的尝试路由均已成功建立。

增益共享

MADI 舞台接口箱

当多台控制台从同一 MADI 舞台接口箱接收信号时，只有一台控制台可以远程控制舞台接口箱的前置放大器。“主”控制台就是连接到舞台接口箱 MADI 输入的控制台（若通过 Blacklight II 连接，则为连接到 Blacklight Concentrator A 端口的控制台）。

Gain Share 功能可让没有前置放大器控制权的控制台（“从”控制台）以数字方式补偿主控制台对模拟信号电平所做的任何更改，从而避免从控制台的总体输入电平发生意外变化：

主控制台工程师确定某通道的初始模拟增益后，从控制台工程师应按下其控制台上的 Gain Share 按钮，启用增益共享。

此后，如果主控制台将模拟前置放大器增益降低 3 dB，从控制台的数字微调会提高 3 dB。如果从控制台未启用 Gain Sharing，则不会对模拟增益变化进行数字补偿。

注意：主控制台上的大幅增益变化（例如启用 Pad 或 High Gain Sensitivity 模式）可能使从控制台在应用补偿微调时产生可听见的电平变化。

位于 Gain Share 按钮顶部的 LED 条颜色表示关闭 Gain Sharing 后信号电平会升高还是降低。灰灯表示尚未应用数字微调偏移（即启用 Gain Sharing 后，模拟增益未发生变化）。

绿灯表示关闭 Gain Sharing 会使电平降低。（模拟增益已调低，因此控制台通过调高数字微调进行补偿；关闭 Gain Sharing 会移除此偏移。）

红灯表示需要谨慎操作，因为关闭 Gain Sharing 会使电平升高。

Gain Share 按钮下方的文本字段显示已应用到该通道的数字微调量。

注意：“主”控制台也有 Gain Share 按钮和数字微调控件。它们不会影响“从”控制台的增益；同样，“从”控制台的 Gain Share 按钮和数字微调也不会影响“主”控制台的增益。
“主”控制台可使用自身的 Gain Share 按钮进行自增益共享，从而即使需要更改模拟增益，也能保持一致的增益结构。如果已设置压缩器、门限器阈值等参数并希望保持这些设置，此功能会很有用。
