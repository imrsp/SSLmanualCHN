通过 SOLSA PC 软件、其他控制台和 Remote Expander 进行远程控制

本节说明如何通过网络连接，从运行 SOLSA 的电脑、 另一台 SSL Live 控制台，或 Remote Expander 对控制台进行远程控制。 Remote Tile 通过 USB 连接到控制台，其专门说明页面如下： Remote Tile。

远程控制台操作面或运行 SOLSA 的计算机可通过以太网连接到主控制台，共享其音频处理能力。

最多可由 2 台设备同时远程控制控制台。

SOLSA、处于 Remote Surface 模式的另一台 SSL Live 控制台以及 Remote Expander 的连接过程基本相同。

SOLSA

SOLSA （用于 PC 的 SSL 在线/离线设置应用）提供离线界面，使用户无需控制台即可准备演出文件。 SOLSA 也可在线远程控制任何 SSL Live 控制台。

处于 Remote Surface 模式的另一台 SSL Live 控制台

如果有多台 SSL Live 控制台，它们可相互远程控制。 这实际上扩展了控制台的操作面。

说明： 此功能会增加可控制音频处理的操作面数量，但不会增加可用音频处理能力。 将控制台配置为 Remote 会禁用其内部音频引擎。

Remote Expander

Remote Expander 是 SSL Live 的专用远程操作面。


与 Live 系列的其他操作面不同，Remote Expander 不具备音频处理能力；它只是连接到 Live 控制台的操作面。 Remote Expander 提供 24 或 36 推子型号，配有主触摸屏和用于外接屏幕的 VGA 输出。

连接到控制台

在控制台上，进入 MENU > Setup > Options > REMOTE 选项卡。

使用 SOLSA 连接

使用 SOLSA 连接

确保控制台在 Surface Settings 部分中设为 Main Surface 模式。如不是，请按住 Main Surface 按钮。

选择一个密码并输入 Password 字段。

将 PC 的网络端口连接到控制台背面的一个 Network 端口。

如需无线连接控制台，可将无线接入点连接到控制台的一个 Network 端口。 然后将 PC 连接到该无线网络。

说明： 确保使用控制台的 Network 端口；Dante 端口不能用于远程操作面控制。

确认控制台 IP 地址与计算机位于同一子网。
控制台的 IP 设置位于 MENU > Setup > Options > Network 选项卡。 更多信息请参阅 Network Options 部分。
PC 的 IP 地址在控制面板的“网络和 Internet”设置中配置。

打开 SOLSA。

在 SOLSA 中，进入 MENU > Setup > Options > Network 选项卡， 然后选择一个 Adapter，或选择 PC 上连接控制台的以太网端口。更改适配器后，请关闭并重新打开 SOLSA。

在 SOLSA 中，进入 MENU > Setup > Options > Remote 选项卡。

按住 Remote Surface。

SSL Live Connections 区域列出网络中所有可见控制台。 找到要连接的控制台。其默认名称以“LIVE-”开头，可在控制台的 System 菜单中更改（MENU > Setup > System / Power > System 选项卡）。

Status 列显示网络中各控制台的连接状态。 点击 Discovered 按钮并从下拉列表中选择 Connect。

一个闪烁的 图标会出现在控制台状态栏中，表示有待处理的连接。

输入密码并按 OK。

远程端的 Status 按钮此时会显示 Waiting for server response。

在控制台上，按 Awaiting authorisation 按钮，再按其中一个 Connect to slot x 按钮。

此时将建立连接。

使用 Disconnect 按钮拒绝连接请求。

使用 Ban 按钮拒绝请求，并阻止该设备再次请求访问。

连接后，主控制台和远程端的 SSL Live Connections 区域都会将对方显示为 Connected OK，两个状态栏都会显示 图标。

要断开连接，请按 Connected OK 按钮并选择 Disconnect。

使用另一台 SSL Live 控制台或 Remote Expander 连接

使用另一台 SSL Live 控制台或 Remote Expander 连接

确保负责音频处理的主控制台在 Main Surface 部分中设为 Surface Settings 部分. 如不是，请按住 Main Surface 按钮。

选择一个密码并输入 Password 字段。

将主控制台背面的一个 Network 端口连接到另一台控制台/Remote Expander 的一个 Network 端口。

说明： 确保使用控制台的 Network 端口；Dante 端口不能用于远程操作面控制。

确认主控制台 IP 地址与另一台控制台/Remote Expander 位于同一子网。
IP 设置位于 MENU > Setup > Options > Network 选项卡。 更多信息请参阅 Network Options 部分。

在远程控制台/Remote Expander 上进入 MENU > 设置 > Options > Remote 选项卡。

按住 Remote Surface。

SSL Live Connections 区域列出网络中所有可见的主控制台。 找到要连接的控制台。其默认名称以“LIVE-”开头，可在主控制台的 System 菜单中更改（MENU > Setup > System / Power > System 选项卡）。

Status 列显示网络中各控制台的连接状态。 点击 Discovered 按钮并从下拉列表中选择 Connect。

一个闪烁的 图标会显示在主控制台状态栏中，表示有待处理的连接。

输入密码并按 OK.

远程端的 Status 按钮此时会显示 Waiting for server response。

在主控制台上，按 Awaiting authorisation 按钮，再按其中一个 Connect to slot x 按钮。

此时将建立连接。

使用 Disconnect 按钮拒绝连接请求。

使用 Ban 按钮拒绝请求，并阻止该设备再次请求访问。

连接后，主控制台和 SOLSA 的 SSL Live Connections 区域都会将对方显示为 Connected OK，两个状态栏都会显示 图标。

要断开连接，请按 Connected OK 按钮并选择 Disconnect。

手动发现控制台

手动发现控制台

从远程操作面/SOLSA 实例自动检测控制台在某些网络中无法工作，例如通过虚拟专用网络（VPN）在广域网（WAN）上连接两端时。 即使在这种情况下，也可以通过在远程操作面中手动输入主控制台的 IP 地址来建立连接。

按照上述说明准备远程操作面，以连接控制台。 如果通过 VPN 连接时远程操作面未检测到控制台，请将主控制台 IP 地址输入 MANUAL DISCOVERY area of the 远程操作面's Remote 选项页面（MENU > Setup > Options > REMOTE）。 控制台 IP 地址可在 Network 选项页面（MENU > Setup > Options > NETWORK）中查看。

确保远程操作面在 远程操作面's Network 选项页面（MENU > Setup > Options > NETWORK）。更改网络适配器后必须重启远程操作面。

在 MANUAL DISCOVERY area of the 主控制台's Remote 选项页面（MENU > Setup > Options > REMOTE）。 默认端口号为 11973。 在 MANUAL DISCOVERY area of the 远程操作面's Remote options 页面.

继续按照上述说明，以常规方式将远程操作面连接到控制台。

也可将远程操作面自动授权到主控制台远程插槽 1，从而无需在主控制台手动授权（前提是输入正确密码）。 按住 Enabled 按钮即可在主控制台 Remote 选项页面的 MANUAL DISCOVERY 区域启用此选项。

连接后，可使用主控制台 Layer Manager 中的 Remote Sync 按钮 （Menu > Setup > Layer Manager）。每个连接到主控制台的远程操作面都对应一个 Remote Sync 按钮。 这些按钮在同步期间会被禁用；如果要同步两个远程操作面，请等待一个完成后再开始另一个。 这些设置会独立于主控制台的设置存储在主 showfile 中。请注意，更新主控制台设置不会自动更新远程操作面，除非再次按相应的 Sync 按钮。

重要说明：
- 控制台不提供任何 VPN 功能。 可能需要第三方 VPN 服务才能在两个远程地点之间建立网络链路。

－此功能通过网络连接提供控制台远程控制；它 does not 包含通过网络在控制台与远程连接之间传输音频的方法。 音频传输必须使用第三方编解码器，并与控制数据连接并行运行。

- 端到端网络延迟（例如通过 ping 控制台测量）应小于 100 ms。 这是控制信号往返控制台的最低延迟，与站点间音频延迟无关；音频延迟取决于所用编解码器。

- Remote connections must 通过高带宽网络建立；每个远程操作面连接控制台时最多需要 25 Mbps 带宽。

- 使用此功能时必须谨慎。 不在现场操作 PA 或监听系统存在风险，因为工程师无法直接、即时地感知场馆内产生的声音。 SSL 不对使用此功能造成的设备损坏或人身伤害承担责任！

远程控制范围

请注意，远程操作面无法访问控制台的某些区域。 其中包括加载 showfiles, 更改 Console Configuration 以及关闭控制台电源。

某些项目始终由所有操作面并行控制。 其中包括所有影响音频处理的功能，例如 Solo、Mute、母线路由、自动化更改和 FX 等。

其他项目可为每个操作面独立配置，包括：

  • Fader Tile 层和 Bank
  • 亮度设置
  • 外接屏幕选择
  • 鼠标和触控设置
  • 操作面锁定
  • 其他设置位于 Console Options 选项卡中，例如控制台时间、日期和键盘区域
  • 电平表响应特性 （不包括电平表信号源）
  • Surface Options 选项卡中的所有项目（区块设置、Query 和 Follow 模式）
  • User Keys

Query 按操作面独立工作，因此在一个操作面上 Query 某条路径不会影响另一个操作面。 也可在两个或更多操作面上 Query 不同信号路径。

操作面网络配置，以及 Automation GUI 和区块锁定控制，也都按操作面独立设置。
