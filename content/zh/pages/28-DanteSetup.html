IP 地址设置

开始配置前，需要了解 SSL Live 控制台包含五个独立的网络适配器。 要与外部 Dante 网络正常配合，必须正确配置以下设备：

  • Dante Control 主网
  • Dante Control 备网
  • Dante Expander Audio 主网
  • Dante Expander Audio 备网
  • Connectivity 网络

如果使用 BLII Bridge 或 X-Light Bridge，还需要配置以下网络设备：

  • BLII/X-Light Bridge Audio 主网
  • BLII/X-Light Bridge Audio 备网

Dante Control 主、备网 - 提供对 Dante 网络的控制，包括创建/取消路由以及控制麦克风增益等输入参数。 两个适配器应分别位于 Dante 主网和 Dante 备网。

Dante Expander Audio 主、备网 - 通过内部 Dante Expander 模块在 Dante 网络上传输音频。Dante Expander 不负责控制路由或舞台接口箱。 两个适配器应分别位于 Dante 主网和 Dante 备网。

Connectivity 网络 - 用于通过 SOLSA、扩展面板、另一个 SSL Live 控制台或 TaCo 应用程序。 可通过控制台背面三个标有 NETWORK 的端口之一接入该网络。 Connectivity 网络是独立于 Dante 网络的网络，Dante 操作不需要该网络。 但是，请务必注意其 IP 地址，如果设置不正确，可能会与您的 Dante 网络发生冲突。

X-Light Bridge Console Link 主、备网 - 通过 X-Light Bridge 在 Dante 网络上传输控制台的 Dante Control 和 Dante Expander 数据。两个适配器应分别位于 Dante 主网和 Dante 备网。

上述网络设备应位于 3 个独立的网络上：

这些网络应包含以下设备。

  • Dante 主网
    • 控制台的 Dante Control Primary
    • 控制台的 Dante Expander Audio Primary
    • BLII Bridge Audio Primary（如果使用 BLII Bridge）
    • X-Light Bridge Audio Primary（如果使用 X-Light Bridge）
    • 其他 Dante 设备的 Primary 连接（例如 Net I/O 舞台接口箱、Dante Virtual Soundcard、第三方 Dante 设备）
    • 运行 Dante Controller 的计算机
  • Dante 备网
    • 控制台的 Dante Control Secondary
    • 控制台的 Dante Expander Audio Secondary
    • BLII Bridge Audio Secondary（如果使用 BLII Bridge）
    • X-Light Bridge Audio Secondary（如果使用 X-Light Bridge）
    • 其他 Dante 设备的 Secondary 连接（例如 Net I/O 舞台接口箱、Dante Virtual Soundcard、第三方 Dante 设备）
  • Connectivity 网络:
    • 控制台 Connectivity 网络
    • 远程模式下的 SOLSA（如果使用）
    • 运行 TaCo 的平板电脑（如果使用）
    • 远程模式下的另一个 SSL Live 控制台（如果使用）

每个网络都应该有自己的子网。同一网络上的每台设备都必须拥有同一子网内的 IP 地址，以便这些设备能够相互通信。

每个网络设备都需要在其连接的网络的正确子网内有一个唯一的 IP 地址。

注意：
Dante Control 和 Dante Expander Audio 均可通过控制台背面的 Dante Expander Primary 和 Secondary 端口访问。 控制台中的内部网络交换机连接这两个独立的网络设备。它们都有单独的 IP 设置。 例如，Dante Control Primary 和 Dante Expander Primary 必须在同一子网内且具有不同的 IP 地址。

Dante Control 和 Dante Expander Audio 也可通过 X-Light 的 Console Link 端口访问。如果要将控制台 Dante Expander 端口和 X-Light 端口接入同一网络，请避免形成网络环路。

要在 SSL Live 控制台上正确配置并控制 Dante，必须配置 Dante Control 和 Dante Expander Audio 网络设备。

要通过 BLII/X-Light Bridge 在 SSL Live 控制台上正确配置并控制 Dante，必须配置 Dante Control、Dante Expander Audio 和 BLII/X-Light Bridge Audio 网络设备。

需要记住的重要事项：

  • 每个设备的 IP 地址必须是唯一的。
  • Dante 主网和备网绝不能进行物理连接。
  • 同一设备上的多个网络适配器不能使用同一子网中的地址。
  • Dante Primary 和 Secondary 地址必须位于不同子网。
  • 当设置为自动配置时，Dante Primary 将使用 169.254.x.x 子网。这是标准 IPv4 协议，与控制台 Connectivity 网口的自动配置机制完全相同。
  • 对于除 SSL Live 控制台外的 Dante 设备（见下文“固定 IP 地址”），Secondary 设置为自动配置时会使用 172.31.x.x 子网。 该范围属于 IPv4 私有地址，但并非专为自动配置保留；将其用于 Secondary 自动配置是 Dante 的特定做法。
  • 除使用不同子网外，Connectivity 网络、Dante 主网和 Dante 备网还应视为三个物理隔离、各自拥有网络硬件的网络。*
  • 默认网关设置必须留空，因为 Dante 当前不使用该设置。**

* 也有例外 - 您可能有一个可配置的网络交换机，可以为多个网络提供服务，并使用 VLAN 进行分隔。 尽管这可行，但保持硬件分离的原因主要与故障转移有关 - 如果这台交换机发生故障，主网和备网会同时中断，失去冗余网络的意义，因此不建议在现场使用中采用这种方式。

**默认网关用于在子网之间路由 IP 数据。 Dante 不支持此功能，输入网关设置会将主子网和辅助子网链接在一起，这是无效的。

SSL Live 建议

SSL Live 应使用固定 IP 地址还是 DHCP？ SSL 推荐：
对于包含 SSL Live 控制台的 Dante 网络，SSL 建议备网上的所有 Dante 设备使用固定 IP 地址， 除非网络中存在 DHCP 服务器。

动态主机配置协议（DHCP）允许服务器自动为预先定义的地址范围内的设备分配 IP 地址。 DHCP 还可以提供有关网关和 DNS 服务器的信息（如果在专用于音频传输的封闭网络中，通常不需要）。

如果控制台的三个控制网络适配器（Connectivity 网络、Dante 主备控制网）中有两个或更多设为 DHCP，而相应网络中没有 DHCP 服务器， 这些适配器可能自动配置到同一个 169.254.xx.xx 子网。这是无效配置，会使 Dante 网络、Connectivity 网络或两者工作异常。

因此，如果所有 Dante 设备都设置为 DHCP，SSL 建议在所有使用的网络上都存在 DHCP 服务器。

另一种方法是为 Dante 主网和 Dante 备网上的所有设备设置固定 IP 地址，以防 Dante Control Primary 和 Secondary 适配器自动落入同一子网。 此时，只要 Dante 主网和备网的固定地址范围不与 Connectivity 网络的 DHCP 地址池重叠，就可以将 Connectivity 网络安全地设为 DHCP； 如果 Connectivity 网络中没有 DHCP 服务器，则也不要让 Dante 主网和备网的固定地址与 169.254.xx.xx 自动配置范围重叠。

如果使用多个 DHCP 服务器（例如每个网络一个），则应将它们配置为在唯一子网内发布地址，以确保它们的范围不会与任何其他网络的固定/DHCP 范围重叠。

如果 Connectivity 网络上没有 DHCP 服务器，请将此适配器设置为与 Dante 主网和 Dante 备网位于不同子网中的固定地址。 将 Connectivity 网络上的所有其他设备设置为同一子网内的固定 IP 地址。有关设置Connectivity网络的更多信息，请参阅 远程操作面 或 TaCo 平板控制。

欲了解更多信息，请参阅 IP 地址冲突警告.


查找设置为不同子网中固定 IP 地址的 Dante 设备

Audinate 网站包含有关如何查找已设置为与计算机不同子网中的固定 IP 地址的设备的详细说明。 www.audinate.com/faq/how-can-i-locate-dante-device-static-ip-address-unknown-subnet

也可以使用免费的 Wireshark 识别网络中设备的 IP 地址。

首先，从他们的网站下载 Wireshark： www.wireshark.org 并在您的计算机上安装该程序。

Wireshark 的工作原理是报告网络上设备的所有 IP 地址。为了简化调查，请将您的计算机直接连接到相关设备上的 Dante 主网适配器。

打开 Wireshark。您计算机的所有网络适配器都会列出。双击用于 Dante 主网的网络适配器。大多数笔记本电脑仅包含一个网络适配器，因此只会显示一个有线网络连接。

网络上的所有设备现在都将按其 IP 地址显示。如果您直接连接到相关设备，则此处仅可见两个 IP 地址：相关设备和您的计算机。

确定哪个 IP 地址是您计算机的 IP 地址。

在 Mac 上，可前往 System Preferences > Network 查看 IP 地址；点击左侧的以太网适配器即可显示 Mac 的 IP 地址。

在 PC 上，打开 Command Prompt，输入 ipconfig 并按 Enter；计算机的 IP 地址显示在 IPv4 Address 旁。

一旦确定 Wireshark 中显示的 IP 地址是您的计算机的 IP 地址，其余的 IP 地址将是相关设备的 IP 地址。

将计算机的 IP 地址设置为与相关设备位于同一子网内，然后该设备将在 Dante Controller 中可见。更改设备的 IP 地址，使它们位于您希望用于 Dante 主网和备网的子网内，并设置计算机的 IP 地址，使其回到您用于 Dante 主网的范围内。

步骤 1：设置 Dante Control IP 地址

本节介绍如何设置 Dante 主备控制网络适配器的 IP 地址。 这些适配器用于控制 Dante 网络，包括建立或取消路由，以及控制话筒增益等输入参数。 两个适配器应分别位于 Dante 主网和 Dante 备网。

在控制台上转到 MENU > Setup > Options > NETWORK 选项卡。

屏幕左侧显示Connectivity网络信息，右侧显示 Dante Control 主、备网信息。

注意：
这些地址用于控制台使用的网络适配器，用于向 Dante 网络发送控制数据或从 Dante 网络接收控制数据， 不是 Dante 音频连接的地址。 Dante Control 网络适配器在 Dante Controller 中不可见，也无法由 Dante Controller 进行配置。

在本示例中，将主控制网络适配器设为：

IP 地址： 10.101.1.1

网络掩码： 255.255.0.0

网关： 0.0.0.0

首先将适配器类型设置为 Fixed，然后双击“IP Address”和“Netmask”框并输入上面的值。 完成此操作后，按住 Apply 按钮并等待网络适配器重新启动。

然后将备控制网络适配器设为：

IP 地址： 10.102.1.1

网络掩码： 255.255.0.0

网关： 0.0.0.0

将适配器类型设置为 Fixed，然后双击“IP Address”和“Netmask”框并输入上面的值。 完成此操作后，按住 Apply 按钮并等待网络适配器重新启动。

在我们的示例中， Connectivity 网络适配器 地址设置应保留为 DHCP。 如果以后需要将其设置为固定 IP 地址，请确保使用与 Dante 网络不同的子网。

步骤 2：设置 Dante Audio IP 地址和名称

要设置控制台 Dante 音频网络设备的 IP 地址，您需要一台运行 Audinate 的 Dante 控制器应用程序的计算机，您可以在此处下载该应用程序： www.audinate.com/products/software/dante-controller

将计算机的网络适配器设置为固定 IP 地址 10.101.10.10，网络掩码为 255.255.0.0。 现在，将计算机的网络端口直接连接到控制台上的 Dante Pri 端口，然后启动 Dante Controller 应用程序。

在 Dante Controller 中找到控制台；本示例中的名称为 L200-Dante-Expander （在初始设置时，此名称将有所不同）。

双击设备名称，在 Device View 窗口中选择 Device Config 选项卡。

在 Rename Device 字段中输入合适的名称，使网络中的用户能在 Dante Controller 中轻松识别该设备。 控制台型号和场地位置通常就足够了（例如“L200-FOH”），除非配置非常大的网络。

在我们的示例中，确保 Dante 设备采样率设置为 96 kHz，如上所示。

接下来选择 Network Config 选项卡。此页面显示控制台的 Dante 音频网络设备的 IP 地址。

在我们的示例中，我们将为主网适配器使用以下 IP 地址设置：

IP 地址： 10.101.2.2

网络掩码： 255.255.0.0

备网适配器使用：

IP 地址： 10.102.2.2

网络掩码： 255.255.0.0

按 Apply 以使更改生效。系统将要求您确认 IP 地址更改，单击 Yes。 现在必须重新启动设备才能使更改生效。点击 Reboot。系统将要求您确认重新启动。点击 Yes.

注意：
对于我们的示例，请确保 虚拟跳线 在 Dante Expander 详细信息视图中值设置为 0 （在 MENU > Setup > I/O > Local/MADI Configuration）。 这样可简化示例。有关配置虚拟跳线的更多信息，请参阅 使用虚拟跳线。

步骤 3：设置 Dante 音频网络和 Dante 控制网络名称

为了轻松识别 Dante 网络上的控制台，控制台使用两个唯一的用户定义名称：

Dante Device Name - 标识控制台的本地 Dante 音频接口（名称在 Dante Controller 中可见并在上面的步骤 2 中设置）

Control ID - 将控制台标识为 Dante 设备控制器（用于 Net I/O 舞台接口箱所有权的名称）

重要：这两个名称不得相同。

Dante Device Name 位于 MENU > Setup > I/O > Local/MADI Configuration > Dante Expander。 当控制台连接到 Dante 网络时，应自动填充此名称。 如果没有自动检测到名称，请双击 Device Name 字段， 并完全按照 Dante Controller 中显示的名称输入。

Control ID 位于 MENU > Setup > Options > NETWORK 选项卡。 下例中的 Control ID 为 LIVE-L200。

Control ID 取自 Console Name。前往 MENU > Setup > System / Power > System 选项卡， 在 Console Name 区域双击名称进行编辑，然后按住 Apply。

请注意：
改变 Console Name 将关闭控制台电源，并且控制台将失去所有 Net I/O 舞台接口箱参数的所有权。 这是因为控制台使用控制 ID（取自Console Name）来拥有自己的舞台接口箱输入和参数。 所有先前拥有的舞台接口箱输入和参数都需要重新拥有。 有关如何拥有输入和舞台接口箱参数的更多信息，请访问 控制网络 I/O 舞台接口箱和所有权

步骤 4：连接 SSL Net I/O 舞台接口箱（和其他 Dante 设备）

现在您已经配置了控制台的 Dante 设置，我们可以开始将网络交换机和其他 Dante 设备添加到网络。

注意：
主网和备网必须始终保持物理分离。 请勿将主网交换机与备网交换机相连，也不要将设备的 Primary 端口接入备网交换机，反之亦然。

所有 SSL 舞台接口箱出厂时均配置为 DHCP。

要在舞台接口箱上设置固定 IP 地址，请将您的计算机（配置为使用 DHCP 地址）直接连接到舞台接口箱并使用 Dante Controller 设置固定 IP 地址（按照步骤 3 中的说明进行操作）。

一个网络上的每台设备都必须拥有同一子网内的 IP 地址，以便这些设备能够相互通信。

因此，对于我们的示例，连接到 Dante 主网的任何设备都必须具有如下 IP 地址：

IP 地址： 10.101.x.x

网络掩码： 255.255.0.0

注意：
我们已经在主网络上使用了 IP 地址 10.101.1.1、10.101.2.2 和 10.101.10.10。这些 IP 地址不能用于网络上的其他设备。

对于 Dante 备网，任何连接的设备都必须具有如下 IP 地址：

IP 地址： 10.102.x.x

网络掩码： 255.255.0.0

注意：
我们已经在备网上使用了 IP 地址 10.102.1.1 和 10.102.2.2。这些 IP 地址不能用于网络上的其他设备。

使用 Dante 控制器或控制台的 Dante I/O 设置页面，将所有连接的舞台接口箱设置为 96 kHz 采样率。这是因为我们在此示例设置过程中之前已将 Dante Expander 模块设置为 96 kHz。 具有不同采样率的两个设备之间无法进行 Dante 路由。因此，要在两个 Dante 设备之间创建路由，这些设备必须以相同的采样率运行。

Dante Expander 模块

SSL Live 控制台可选装 Dante Expander 模块。该模块包含 Dante Brooklyn II 模块，用于与 Dante 网络建立音频连接。

控制台上可用于路由的通道数量将根据控制台和 Dante 网络采样率而有所不同。 如果 Dante 网络以与控制台不同的采样率或在不同的时钟域中运行，请启用 SRC In 按钮。 这将在控制台和 Dante 网络之间转换传入和传出音频的采样率。下表总结了可用的采样率组合 （请注意引用的数字适用于每个方向，例如 32 = 32 个输入 和 32 个输出）：

控制台采样率 Dante Expander 采样率 Dante Expander 采样率转换 Dante 输入和输出的数量
96 kHz 192 kHz SRC In 16
96 kHz 96 kHz SRC Out 32
96 kHz 48 kHz SRC In 32
48 kHz 192 kHz SRC In 16
48 kHz 96 kHz SRC In 32
48 kHz 48 kHz SRC Out 64
注意： 除非 SRC In 按钮已按下，您必须设置 Dante Expander 的采样率以匹配控制台的采样率。 这可以通过运行 Dante Controller 软件且与控制台的 Dante Expander 连接到同一网络的计算机来完成。或者，从 Live v6.2 开始，可以在 Dante 配置页面的已连接设备列表中查看和更改控制台扩展器的采样率。

Dante Expander 模块同时支持双域路由和虚拟跳线。 欲了解更多信息，请参阅 Dante 双域路由 和 Dante 虚拟跳线.

Net I/O BLII/X-Light Bridge

Net I/O BLII Bridge 或 X-Light Bridge 可连接到 SSL Live 控制台，以 96 kHz 或 48 kHz 提供额外的 256 个 Dante 输入和输出。

SSL Live 控制台可以同时利用 BLII/X-Light Bridge 的 Dante I/O 和 Dante Expander 模块。

BLII/X-Light 网桥支持双域路由和虚拟跳线。欲了解更多信息，请参阅 Dante 双域路由, Dante 虚拟跳线 和 BLII 与 X-Light 网桥.
