网络选项

这些设置涉及控制台本地网络和 Dante IP 配置。

网络选项位于 MENU > 设置 > Options > NETWORK 选项卡。

本页包含以下控制台网络设备的 IP 设置：

连接控制网络

用于通过 SOLSA、远程控制扩展模块、另一台 SSL Live 控制台或 TaCo 远程控制控制台。 还用于与 DAW 和其他 ipMIDI 设备通信。 此网络通过控制台背面三个标有 NETWORK 的端口之一接入。

要将设备连接到 Connectivity 网络，该设备的 IP 地址必须与本页所列 Connectivity 网络位于同一子网。

Dante 主控制网络 - 提供 Dante 网络的主控制连接，包括建立或取消路由，以及控制麦克风增益等输入参数。

Dante 备控制网络 - 提供 Dante 网络的备控制连接。

要在控制台上使用 Dante，控制台 Dante 控制网络的主、备 IP 地址必须分别与 Dante 网络中的其他设备处于相同子网。 Dante 主网络和备网络必须使用不同子网。

控制台的三个网络设备（Connectivity、Dante Primary Control 和 Dante Secondary Control）的 IP 地址必须分别位于不同子网。

根据需要，为每个网络选择 DHCP 或 Fixed。 双击任意 IP Address, Netmask 或 Gateway 字段可调出屏幕键盘，以输入固定地址。 正确输入设置后，按住 Apply.

请注意，屏幕上的绿色 LED 只表示网络已连接，并不表示 IP 设置正确。

请注意： 如果这些控制台网络设备中的任意一个设为 DHCP ，且该网络中没有 DHCP 服务器，IP 地址将自动配置在 169.254.x.x 子网内。 如果多个控制台网络设备设为 DHCP 且这些网络中均无 DHCP 服务器，则这些网络设备都会自动配置在 169.254.x.x 子网内。 这会使网络互相连通，属于无效配置。 因此，SSL 建议在没有 DHCP 服务器时使用 Fixed IP 地址。

有关 Dante IP 配置的更多信息，请参阅 Setting Up Dante.

有关将远程操作面或 TaCo 连接到 Connectivity 网络的更多信息，请参阅 远程操作面 and TaCo 平板控制.

IP 地址冲突警告

Network Options 中新增了 IP 地址冲突警告（MENU > Setup > Options > NETWORK 标签页） 中新增了 IP 地址冲突警告，用以在控制台上的多个网络适配器处于同一子网时予以提示。

需要更改 IP 地址的适配器会以红色突出显示。

同时还会显示状态消息。

状态栏中也会显示网络错误图标。

当某个适配器的 IP 地址被改为与另一适配器处于同一子网时，会出现警告。 如果出现这些警告消息，说明控制台网络适配器的当前配置无效，必须更改相应适配器的 IP 地址以解决冲突。

警告按以下优先级排列：

  1. Dante Control Primary
  2. Dante Control Secondary
  3. Connectivity Network

例如，如果 Connectivity 适配器和 Dante 主网均被设置在 169.254.x.x 子网中， 警告会显示在优先级较低的 Connectivity 适配器上。

IP 地址冲突解决后，网络错误图标将消失，并显示以下状态消息：

SNMP 日志记录

简单网络管理协议（SNMP）用于记录网络中设备的信息。 要使用此功能，控制台的 Connectivity 网络中必须有外部 SNMP“管理器”服务。

填写 Username、Authentication 和 Privacy 字段以配置控制台的 SNMP 代理服务，并注意各字段的最短字符数。

使用各字段右侧的 Protocol 按钮，设置 Authentication 和 Privacy 的加密方案。

要启动控制台的 SNMP 代理服务，请启用 SNMP Configuration 部分中的 Enabled 按钮。关闭该按钮即可停止服务。

使用 Export MIB 按钮可导出一个 MIB 文件，其中包含启用 SNMP 时控制台公开的参数。 可将该文件导入 SNMP 管理器数据库，以便配置。

控制台会报告以下状态：

  • 控制台 Connectivity 控制网络状态
  • 控制台 Dante 主网络状态
  • 控制台 Dante 备网络状态

Dante Domain Manager 登录

控制台可连接到由 Dante Domain Manager（DDM）管理的网络域。 控制台必须与要进行路由或控制的 I/O 设备登录到同一个域。

重要说明： 控制台的本地 Dante 扩展模块（Brooklyn II）模块以及所有 BLII 或 X-Light Bridge，必须与 I/O 和控制台控制软件加入同一个域。 此操作通过 Dante Domain Manager 完成。

要将控制台连接到 Dante Domain Manager 服务器，请在 DDM Sever Name/IP Address 字段中输入服务器的 IP 地址或主机名。 在相应字段中输入 DDM User Name 和 Password。连接期间，DDM Server Connection Status 指示灯会闪烁绿色；成功连接到 DDM 服务器后，指示灯变为常亮绿色。

说明： DDM 用户账户必须具备路由权限，因此应在所用域中设为 Operator 或 Administrator 角色。此设置在 DDM 中完成。

使用 Selected Domain 部分，从 DDM 服务器上配置的域列表中选择一个域。 未加入特定域的任何 I/O 设备都会自动放入 <unmanaged> 域，以便发现、路由和控制。 如果网络中配置了 DDM，建议将设备加入某个域。 更改 DDM 设置后，建议重启控制台。

Current Domain 字段显示控制台当前所在域的名称。 如果 DDM 服务器权限发生变化，导致控制台从 Selected Domain 中移除，此处显示的域可能与 Selected Domain 不同。

如果连接 DDM 域失败，控制台会自动重试数次。 也可使用 Attempt Reconnection 按钮手动尝试重新连接（例如确认凭据正确后）。
