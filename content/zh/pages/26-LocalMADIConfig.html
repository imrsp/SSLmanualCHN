I/O 设置：Local/MADI 配置

Local/MADI Configuration（本地/MADI 配置） 用于定义本地连接和 MADI 连接的 I/O 名称及设置。这些设置会随演出文件保存。

要打开此菜单，请前往 Menu > 设置 > I/O，然后点击 Local/MADI Configuration。

注意：上面的屏幕截图来自 L650。其他 SSL Live 控制台型号的 I/O 数量有所不同，但原理相同。

每一列代表一种不同的 I/O 卡类型：

  • Local 显示 Internal（本地） I/O、FX Loop MADI 光纤和可选的 Dante 模块。
  • MADI 显示控制台背面的 BNC/SC 光纤 MADI 端口。这些端口按四个或八个一组排列，列标题中以字母（A、B 或 C）标识，并与控制台后面板的标注一致。
  • BLII 或 X-Light 表示可选的 Blacklight II/X-Light 端口，用于连接 Blacklight II to MADI Conccentrator、Net I/O BLII Bridge 或 X-Light Bridge。最多可安装两个 BLII 端口，列标题和控制台后面板上分别以 1 或 2 标识。
提示：点击列标题可查看各区域在控制台背面位置的图示。

配置 BLII/X-Light 端口

Blacklight II (BLII) 和 X-Light 是 SSL 专有的高带宽数字音频传输协议。 BLII 可通过一根或一对冗余的 Neutrik OpticalCON Duo 多模光纤（也可特别订购单模光纤选件），双向传输 256 个 96 kHz 或 48 kHz 音频通道。MADI 舞台接口箱的控制数据也通过 BLII 传输。

X-Light 可通过一根或一对冗余的 Neutrik OpticalCON Quad 多模光纤，双向传输 256 个 96 kHz 或 48 kHz 音频通道。控制台的 Dante 控制信号和本地 Dante Expander 音频也通过 X-Light 传输。

要将 SSL Live 控制台上的 BLII/X-Light 端口用于 MADI 设备和 SSL MADI 舞台接口箱，必须使用 BLII to MADI Concentrator。 位于舞台端的 BLII to MADI Concentrator 随后用于将标准同轴 MADI 分配至模拟及 AES/EBU 舞台接口箱、第二台 SSL Live 控制台或其他 MADI 设备。

在非加固固定安装场合/非巡演级固定安装场合中，要将 SSL Live 控制台上的 BLII/X-Light 端口用于 Dante 设备及 SSL Net I/O Dante 舞台接口箱，建议使用 Net I/O BLII Bridge。 与控制台放置在一起的 BLII Bridge 用于扩展控制台的 Dante I/O 数量。

在巡演级加固场合中，要将 SSL Live 控制台上的 BLII/X-Light 端口用于 Dante 设备及 SSL Net I/O Dante 舞台接口箱，建议使用 Net I/O X-Light Bridge。 位于舞台端的 X-Light Bridge 用于扩展控制台的 Dante I/O 数量。

要配置 BLII/X-Light 端口，请在控制台上前往 MENU > 设置 > I/O > Local/MADI Configuration。

连接 BLII to MADI Concentrator/BLII Bridge/X-Light Bridge 时，如果相关 BLII/X-Light 端口中尚未配置设备，所连接的设备将被自动检测并配置。

如果相关 BLII/X-Light 端口中已配置设备，则需要手动配置该端口。

配置 BLII/X-Light 端口：

  1. 点击相应端口的列标题以查看该端口的设置。
  2. 在端口的详细视图中按住 Physical Port Selection:，然后选择 Blacklight II（用于 BLII Concentrator 或 Net I/O BLII Bridge）或 X-Light（用于 X-Light Bridge）。 请注意，对于 L450、L550 和 L650，X-Light 始终连接到第二个 BLII/X-Light 端口（I/O 页面最右侧一列）。
  3. 按住 Configure Port For:，并在需要时选择所需设备。

注意：同时安装 BLII 和 X-Light 端口的 L100、L200 和 L350 控制台，一次只能使用其中一种端口类型。
 警告：
重新配置 BLII/X-Light 端口时请谨慎，因为现有路由将被取消。
- 如果将 BLII 端口从 BLII-MADI Concentrator 重新配置为 BLII Bridge，控制台与通过 BLII-MADI Concentrator 连接的设备之间的所有现有路由都将被取消。
- 如果将 BLII 端口从 BLII Bridge 重新配置为 BLII-MADI Concentrator，则无论路由经过 BLII Bridge 还是 Dante Expander 模块，所有 Dante 双域路由都将被取消。 已建立到 Dante Expander 模块的虚拟跳线路由将保留。

BLII/X-Light 端口配置存储在演出文件中。

如果使用 BLII to MADI Concentrator，该设备现已配置完毕，可以使用。

如果使用 Net I/O BLII Bridge 或 X-Light Bridge，请参阅 使用 BLII 和 X-Light Bridge。

MADI 和经由 BLII 的 MADI

以下说明适用于通过 MADI 或 BLII to MADI Concentrator 连接的设备。

每个端口都会自动检测 SSL Live 舞台接口箱。MADI 和 Blacklight 端口提供完全相同的配置选项。

注意：部分 MADI 和 BLII 列可通过在屏幕上向上或向下滑动来滚动。

也可以在连接 I/O 之前预先配置（例如使用 SOLSA 离线软件）：

点击端口顶部的深灰色方框，然后选择所需的设备类型。

注意：要更换为其他设备，需要在 MADI 和 BLII 端口列表下方的详细信息窗格中确认更改。

如果定义的设备与端口检测到的设备不匹配，方框背景将变为红色；请检查连接或手动更改设备类型。在 MADI 和 BLII 端口列表下方的详情窗格中确认要使用的设备。

注意：如果已基于原始配置完成进一步设置后再更换设备，之前的设置可能不再有效。

设备选项如下：

  • None - 未连接任何设备
  • Generic MADI - 任意第三方 I/O
  • ML 32.32 - 32 路模拟 I/O
  • ML I.32 - 仅 32 路模拟输入
  • ML O.32 - 仅 32 路模拟输出
  • D 32.32 - 16 对 AES I/O
  • Live Recorder - 连接 MADI 多轨录音机以用于排练模式；此选项已在 V5.2 中移除，应改用 Generic MADI。  
注意：如果控制台以 96 kHz 运行，任何通用 MADI 设备都必须支持 96 kHz 64 通道模式（96 kHz 时为 32 通道）的帧格式（非传统帧格式）。

每个端口编号下方的两个 LED 指示该端口的锁定和冗余状态：

如果上方 LED 为绿色，表示端口的 I/O 单元已正确锁定；如果为红色，则表示未锁定。

如果下方指示灯亮起，表示该端口属于一个冗余端口对（见下文）。

注意：如果使用 16 端口 Blacklight MADI Concentrator，将有四个指示灯；每个冗余硬件连接对应一对。

冗余

每个 MADI 端口都可作为冗余端口对的一部分运行。端口始终按奇偶配对，例如端口 1 始终与端口 2 配对。

要定义冗余端口对：

点击其中一个端口的方框，打开其详细信息对话框。

点击其左侧的 Redundancy Status 按钮。按钮将亮起，表示该端口现已属于冗余端口对；两个端口周围会出现青色边框，表示二者已链接。两个端口下方的状态灯指示当前活动端口：绿色表示活动端口，红色表示备用端口。

第一个端口的设置现在将应用于这一对端口。

注意：
从控制台到任意 Blacklight II.MADI Concentrator 的光纤连接始终支持冗余，只需接入第二根光纤。此处的冗余是指从控制台或 Blacklight II Concentrator 到后续设备的 MADI 冗余。

如果使用 16 端口 Blacklight II.MADI Concentrator，每个端口始终支持冗余，因此无需开启 Redundancy Status 按钮。

连接 16 端口 Blacklight II.MADI Concentrator 后，每个端口会显示四个状态灯。左侧一对对应下排 MADI 接口；右侧一对对应上排 MADI 接口。与 8 端口版本相同，上方指示灯表示 MADI 锁定状态（绿色 = 已锁定，红色 = 未锁定），下方指示灯表示哪个冗余端口处于活动状态（绿色 = 活动，红色 = 备用）。

MADI 舞台接口箱端口配置

点击某个端口或连接的方框，会在页面下半部分的 Detail 对话框中显示该端口的设置。

对于舞台接口箱，右侧的页面按钮（Setup、Inputs 和 Outputs，本地 AES I/O 则为 AES I/O）可用于逐页查看端口设置：

设置：

双击黑色的名称框会弹出键盘，可在其中输入端口名称。
Set Names 按钮可用于为设备选择预设名称。请注意，这不会影响各个输入和输出的名称，只会更改设备本身的名称。

Output Trim 滑块以控制台的 Operating Level 为基准，对舞台接口箱的所有模拟线路输出施加全局电平微调 （在 Menu > Setup > System / Power > System 选项卡中设置）。请注意，这不会影响可选的麦克风分配输出（如已安装）；这些输出始终为麦克风电平。

Clear Attention 按钮会清除本地 I/O 上所有 Att 指示灯。页面其余部分显示格式和时钟信息。

输入和输出：

点击 Inputs 或 Outputs 按钮，会显示舞台接口箱中的所有独立连接。每个连接的方框显示其名称和主要设置。使用 Show Settings 和 Show Routes 按钮，可在前置放大器设置与路由信息之间切换显示。

点击连接方框后，屏幕右侧会显示相关控件。

为连接命名：

  • 点击连接方框；其编号（或名称）会出现在详细信息显示区右上角的名称框中。
  • 双击名称框，使用弹出的键盘输入新名称。
  • 点击 Use Default 会将连接名称恢复为默认名称。
  • 也可以按 Set Names 按钮选择一组名称，应用于舞台接口箱中的所有输入或输出。请注意，对输入使用 Set Names 不会影响输出或舞台接口箱名称，反之亦然。

控制前置放大器（仅限模拟输入）：

  • 点击连接方框；所选输入的控件会显示在 Detail 显示区右侧。
  • 使用 Show Gain 按钮，在显示模拟增益和其他可控参数之间切换。
  • 使用挂锁按钮解锁控件以进行修改。

AES I/O

所有 AES I/O 均配有采样率转换器（SRC），并且默认启用。

点击 AES I/O（Internal）或 Inputs/Outputs（D32.32）按钮，会显示设备内所有独立 AES 连接。每个连接的方框显示其名称和主要设置。使用 Show Settings 和 Show Routes 按钮，可在 SRC 设置与路由信息之间切换显示。

点击连接方框后，屏幕右侧会显示相关控件。

选择 AES 输入时，详细信息区域右上角会显示一个 SRC In 按钮。控制台支持下表 Input Rate 列中所列的输入采样率，并会将传入音频转换为 96 kHz，以供 Live 系统使用。

下表还列出了 AES/EBU 输出可用的采样率。输出连接还有一些附加控件。

注意：倍频控件（x1、x2 和 x4）以 44.1 或 48k 的基础采样率为基准，而不是以控制台的工作采样率（48 或 96k）为基准。

启用 SRC In 并将输出时钟设为 In 后，AES 输出将以对应 AES 输入作为时钟源，采样率为基础采样率（44.1 或 48k）的 1、2 或 4 倍。

将输出时钟设为 Int 时，会使用控制台作为时钟源。也可以将其设为该基础采样率（48k）的 x1、x2 或 x4。

配置其他 I/O

其他 I/O 端口也提供详细信息对话框：

内置 I/O

Internal 详细信息对话框提供 ANA OUT Trim（模拟输出微调） 滑块， 该滑块以控制台的 Operating Level（工作电平） 为基准，对所有本地模拟输出施加全局电平微调 （在 Menu > Setup > System / Power > System 选项卡中设置）。 Clear Attention 按钮会清除本地 I/O 上所有 Att 指示灯。

窗口右侧为每个本地 AES 连接提供一个方框。点击这些方框可访问主要 AES 设置， 如上文 AES I/O 中所述。

其余 I/O 详细信息对话框仅显示有关卡或端口的主要信息。

Dante

Live 控制台还可安装 Dante Expander模块。它可为控制台增加最多 64 路 Dante I/O（取决于采样率；见下文）。

在 I/O 页面最左侧一列中选择 Dante Expander 模块。屏幕下部会显示 Dante Expander 的选项。 Recorder ID 按钮将在下一节说明：多轨录音与播放。选择 Free Route 按钮， 可在控制台路由系统中自由路由 Dante I/O。

控制台上可用于路由的通道数会随控制台和 Dante 网络的采样率而变化。 如果 Dante 网络使用的采样率或时钟域与控制台不同，请启用 SRC In 按钮。 这会对控制台与 Dante 网络之间的输入和输出音频进行采样率转换。下表概括了可用的采样率组合 （请注意，所列数量适用于每个方向，例如 32 = 32 路输入和 32 路输出）：

控制台采样率 Dante Expander 采样率 Dante Expander 采样率转换 Dante 输入和输出数量
96 kHz 192 kHz SRC In 16
96 kHz 96 kHz SRC Out 32
96 kHz 48 kHz SRC In 32
48 kHz 192 kHz SRC In 16
48 kHz 96 kHz SRC In 32
48 kHz 48 kHz SRC Out 64
注意：除非已启用 SRC In 按钮，否则必须将 Dante Expander的采样率设为与控制台采样率一致。 可在与控制台 Dante Expander位于同一网络、运行 Dante Controller 软件的计算机上进行设置。或者，从 Live v6.2 开始，也可在 Dante Configuration 页面的 Connected Devices 列表中查看和更改控制台 Expander 的采样率。

启用 Dante SRC 时，不应让控制台从 Dante 网络获取时钟。有关与 Dante 网络之间时钟设置的更多信息， 请参阅 系统时钟设置 一节。

有关设置 Dante、Net I/O 舞台接口箱或 BLII/X-Light Bridge 的更多信息，请参阅 I/O 设置：Dante。

AES67

AES67 是 IP 音频网络的互操作标准。控制台的 Dante 接口支持 AES67； 可在 Dante Controller 中设备的 AES67 Config 选项卡启用。 启用后，可使用 Dante Controller 创建 AES67 流，将音频传输至其他支持 AES67 的兼容设备。 有关创建 AES67 流的信息，请参阅 Audinate Dante Controller 手册。
