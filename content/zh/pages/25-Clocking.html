时钟与同步

控制台和所有舞台接口箱必须共用同一个数字时钟同步参考源。

通常应使用 Live 的内部时钟为控制台和舞台接口箱系统提供时钟。除非特定应用确有需要，否则无需外部时钟。任何外部时钟的采样率都必须与 Live 控制台当前运行的采样率一致（或使用 PAL 25/NTSC 29.97 视频信号）。

注意：如果使用外部时钟，建议只将控制台连接至时钟源，并让舞台接口箱从控制台的 MADI 或 Blacklight 数据流获取时钟。如果外部时钟源丢失，控制台会切换至下一个可用时钟源（例如内部时钟），系统其余部分仍能保持同步。因此，不建议将外部时钟分别连接至各舞台接口箱，否则在没有公共同步源时，各设备会各自独立自由运行。

时钟源

控制台可使用内部时钟，也可通过视频、AES、字时钟、MADI、Blacklight II（双 SSL 控制台系统）或 Dante 获取外部时钟。 请在 Menu > Setup > Options > SYSTEM 页签中选择。

可配置 Main（主）和 Backup（备）时钟源。如果主时钟源丢失，控制台会自动切换至备时钟源。如果备时钟源也丢失，控制台会改用内部时钟。

使用箭头按钮打开 Main 或 Backup 选择菜单。选择所需时钟源，然后按住 APPLY 按钮。 确认 Current（当前时钟）所示来源与预期时钟源一致。

如果指定的时钟源丢失，请先恢复该时钟源，再使用 Reclock 按钮尝试重新锁定至主/备时钟源。

主屏幕顶部的状态栏会显示时钟源变更消息以提醒操作员。状态栏图标还会指示当前时钟是否已偏离所定义的 主 时钟源：

橙色：主 时钟源丢失，当前使用 备 时钟源。
红色：主 备 时钟源均丢失，当前使用内部时钟。
从 Blacklight II 集线器获取时钟

可将两台控制台连接至 Blacklight II 集线器，共用一组舞台接口箱。 “主”控制台（连接至 Blacklight 集线器的 A 端口）会将其时钟分配给 Blacklight 系统 以及与其连接的所有舞台接口箱和控制台。因此，“从”控制台（连接至 Blacklight 集线器的 B 端口） 应将时钟源设为控制台后面板上对应的 BLII 端口。

主控制台应通过 Blacklight 分配其内部时钟，或分配上述某个外部时钟源， 但不能选择由其担任主设备的任何 Blacklight 端口。 选择 BLII 时钟源按钮后会显示一组子按钮。 从控制台应使用 Blacklight II 选项。 但是，如果外部 MADI 源连接至 Blacklight 集线器上相应的 MADI 输入，主控制台可以使用某个 MADI 选项。 例如，该信号可以是提供时钟的第三方设备所输出的 MADI 馈送，也可以是接收字时钟信号并设置为从外部字时钟输入获取时钟的 SSL 舞台接口箱。（请注意，字时钟不能直接连接至 BLII 集线器的 MADI 端口。）

从本机 Dante Expander 端口获取时钟

Dante 使用自身的时钟选举流程确定 Dante 网络中最合适的主时钟，并会自动选出主时钟。 有关 Dante 时钟选举流程的更多信息，请参阅 Audinate 网站：dev.audinate.com/GA/dante-controller/userguide/webhelp/#clock_synchronization.htm。

要手动选择主时钟，请将该设备设为 Preferred Leader（首选主时钟）。 为此，请打开 Dante Controller 并单击 Clock Status 页签。 为所选主时钟勾选 Preferred Leader 复选框。

如果网络中有多台设备被设为 Preferred Leaders，Dante 时钟选举流程会自动从这些 Preferred Leaders 中选出一台主时钟。

如果主时钟设备的状态发生变化，或更合适的主时钟设备上线，Dante 网络会重新执行时钟选举步骤，以确定最适合该网络的主时钟。

如果未使用 Dante 网络冗余，请使用主连接而非备连接，以确保准确同步。

SSL 建议：SSL 建议将控制台设置为从 Dante 网络获取时钟（从属模式），以利用 Dante 时钟选举流程。
将控制台设置为 Dante 网络的从属设备

在此配置中，控制台以外的某台 Dante 设备担任主时钟。控制台及网络中的所有其他设备均跟随该主时钟。

在 Dante Controller 中进入 Clock Status 页签。 如果希望指定主时钟设备，请为该设备勾选 Preferred Leader。 Dante 有自己的时钟选举流程，因此并非必须设置 Preferred Leader。 在本示例中，请确保网络中所有其他设备均未勾选 Preferred Leader 和 Sync to External （包括控制台的 Dante Expander 以及适用时的 BLII/X-Light Bridge）。

在控制台上进入 MENU > Setup > Options > SYSTEM 页签。 在 SYSTEM CLOCK 区域中，选择 Dante Expander 作为 Main 时钟源，然后按住 APPLY。 确认 Current 时钟源字段显示 Dante Expander。

所选 Preferred Leader（如已设置）现为包括控制台在内的整个网络的主时钟。 控制台现从其 Dante 扩展卡模块获取时钟。Dante 扩展卡模块从 Dante 网络中的 Preferred Leader 获取时钟。 网络中的所有其他设备也都从 Dante 网络中的 Preferred Leader 获取时钟。

注意：
如果在控制台上选择 Dante 作为时钟源， 当 Dante 网络时钟丢失时，控制台的时钟备份机制不会回退至其他时钟源。 Dante 扩展卡模块内置时钟；根据 Dante 时钟选举流程，该时钟会成为 Dante 网络的主时钟， 直到网络中另一台设备被识别为更合适的主时钟。 重要：
如果 Dante 的 SRC In 已启用，不建议让控制台从 Dante 获取时钟。
不建议将通过 MADI 或字时钟从舞台接口箱获取时钟的控制台用作 Dante 网络的主时钟源。
将控制台设置为网络主时钟

在此配置中，控制台是网络主时钟，网络中的所有其他设备均跟随控制台时钟。

要进行此设置，请在控制台上进入 MENU > Setup > Options > SYSTEM 页签。 在 SYSTEM CLOCK 区域中，选择 Int（控制台内部时钟）作为 Main 时钟源，然后按住 APPLY。 确认 Current 时钟源字段显示 Int。

在 Dante Controller 中进入 Clock Status 页签。 为控制台的 Dante Expander 勾选 Preferred Leader 和 Enable Sync To External 复选框。

确保网络中所有其他设备均未勾选 Preferred Leader 和 Sync to External。

控制台内部时钟现已成为 Dante 网络的主时钟。控制台的 Dante 扩展卡模块从控制台内部时钟获取时钟，网络中的所有其他设备则跟随 Dante 扩展卡提供的时钟。

重要：
由于 Dante 的时钟选举流程，之后连接至网络、同样设为 Preferred Leader 且 MAC 地址低于控制台的任何设备，都会被提升为网络主时钟。 在这种情况下，控制台时钟与 Dante 网络时钟将不再同步，可能导致音频中断。因此 SSL 建议不要使用此配置，因为它不提供网络时钟冗余。
通过 BLII 或 X-Light Bridge 获取时钟
重要：如上一节所述，SSL 建议将控制台设为从 Dante 网络获取时钟（从模式），以利用 Dante 时钟选举流程。 即使使用 BLII/X-Light Bridge，也应按上述方式通过本机 Dante Expander 端口完成此设置。 只要控制台和 BLII/X-Light Bridge 都持续从 Dante 网络获取时钟，两者就会保持同步。

将控制台设置为 Dante 网络的从设备

为获得最可靠的时钟方案，SSL 建议在使用 BLII 或 X-Light Bridge 时，将控制台设置为 Dante 网络的从设备。

在此配置中，BLII/X-Light Bridge 和控制台（通过本机 Dante Expander）均设置为从 Dante 网络获取时钟。 请参阅上一节，将控制台配置为从本机 Dante Expander 获取时钟。

还必须将 BLII/X-Light Bridge 配置为从 Dante 网络获取时钟。 确保其 Preferred Leader 和 Enable Sync To External 复选框在 Dante Controller 的 Clock Status 页签中均未勾选。

这样可确保 BLII/X-Light Bridge 不从控制台获取时钟，而是设置为接收 Dante 网络时钟。

如果配置正确，BLII/X-Light Bridge 上的 LED 状态应如下：

BLII Bridge LED X-Light Bridge LED LED 含义 正确配置时的状态
BL S XL S 通过 BLII/X-Light 连接从控制台接收同步（时钟） 熄灭
SRC 当前未使用 熄灭
NET S 从网络接收同步（时钟） 绿色常亮
GM N/A 设备被选为网络 Grand Master（主时钟） 熄灭

（GM LED 位于 BLII Bridge 前面板 Dante 端口的右侧。）

如果没有同步 LED 亮起

如果系统已按上述方式正确配置，但 BL S/XL S 和 NET S 同步 LED 均熄灭， 则 BLII/X-Light Bridge 很可能已被选为网络主时钟（即网络中未找到其他 Preferred Leader）。这是有效的时钟方案。 BLII/X-Light Bridge 会使用其内部时钟为网络提供时钟，控制台再从网络获取时钟。此时 BLII Bridge 的 GM LED 会亮起。 Dante Controller 中的 Clock Status 页面会指示当前选出的主时钟。

如果 LED 状态与上述说明不同，请查阅 使用 Dante Bridge 中的完整表格。

设置采样率

Live 系统可在 96 kHz（推荐）或 48 kHz 采样率下运行。使用 System 页面 （MENU > Setup > System / Power > System 页签）中的 96 kHz 和 48 kHz 按钮更改控制台采样率。舞台接口箱也必须改为与控制台采样率一致；见下文。

除 FX Loop 和可选 Dante 模块接口外，将采样率降至 48 kHz 不会增加总输入/输出数量；每个 MADI 端口在 48 kHz 下承载 64 个通道，但奇偶编号的 MADI 端口对始终互为冗余（双卡 Blacklight 集线器上的偶数编号端口会被禁用）。

重要：更改控制台采样率会中断音频，并因需要更改舞台接口箱配置而导致路由断开。建议在切换采样率时将所有输出静音，并在切换后继续静音 30 秒。

重要：更改采样率后，控制台必须重新启动。

MADI 舞台接口箱

默认情况下，舞台接口箱设置为从传入的 MADI 数据流获取时钟。这是推荐设置。其他舞台接口箱时钟同步选项包括内部时钟或外部字时钟。

可通过各舞台接口箱后面板上的实体设置控件设定时钟源和采样率。当前时钟源显示在 SETUP 区域右侧：如果两个时钟 LED 均未亮起，则舞台接口箱正使用自身内部时钟运行。

注意：字时钟输出会根据舞台接口箱的时钟源设置，发送该舞台接口箱当前使用的时钟信号（即该输出并非直通环回）。

要更改时钟源，请同时按住 REMOTE 和 CLOCK，直到挂锁指示灯呈绿色闪烁。按 CLOCK，直至 MADI 亮起。颜色含义如下：

  • 红色：未检测到来自主控制台的 MADI。
  • 红/绿闪烁：已锁定一个主 MADI 端口（非冗余布线）。
  • 绿色：主 MADI 端口 1 和 2 均已锁定（冗余布线）。

要更改舞台接口箱的采样率，请按上述方式解锁控件，然后使用 RATE 按钮循环选择可用选项。

注意：目前仅支持 48 x1 和 48 x2（96 kHz）时钟速率。

以 48 kHz 采样率运行的舞台接口箱可在单条 MADI 数据流上进行菊花链连接，从而利用 MADI 协议的全部 64 个通道。解锁控件后，使用 ID 按钮在通道 1-32 和 33-64 之间切换。

  • 将控制台或 Blacklight 集线器的 MADI Out 连接至第一台舞台接口箱（ID 1-32）的 MADI In。
  • 将第一台舞台接口箱的 MADI Out 连接至第二台舞台接口箱（ID 33-64）的 MADI In。
  • 将第二台舞台接口箱的 MADI Out 连接至控制台或 Blacklight 集线器的 MADI In。
  • 如果采用冗余连接，请对第二组 MADI 端口重复上述步骤。

更多信息请参阅系统示例部分。

注意：如果以 96 kHz 采样率（48 x2）运行，ID 字段必须设为 1-32。

几秒后，控件会恢复为锁定状态。

AES/EBU

所有 AES/EBU 端口均提供采样率转换器（SRC）。这些转换器默认通过控制台的 I/O 菜单启用（MENU > Setup > I/O）。

在 I/O 页面中选择本机 I/O 或 D32.32 舞台接口箱，然后在屏幕下部选择要进行采样率转换的具体 AES/EBU 输入或输出。

对于输入，所选端口会显示 SRC In 按钮。控制台支持下表 input Fs 列中列出的输入采样率，并会将传入音频转换为 96 kHz（或 48 kHz）。

注意：倍频控件（x1、x2 和 x4）以 44.1 或 48k 的基础采样率为参照，而不是以控制台的工作采样率（48 或 96k）为参照。

启用 SRC In 并将输出时钟设为 In 后，AES 输出会从相应的 AES 输入获取时钟，速率为基础采样率（44.1 或 48k）的 1、2 或 4 倍。

将输出时钟设为 Int 时，将使用控制台作为时钟源。也可将其设为基础采样率（48k）的 x1、x2 或 x4。

提示：将输出 SRC 设为 In 后，该 AES 端口的输出会从相应输入获取时钟，而不是使用控制台时钟。如果外接设备难以锁定外部时钟，此设置会很有用。它允许外部设备使用内部时钟，同时让 Live 控制台的 AES SRC 锁定至外部设备，使该设备自行提供时钟，从而避免时钟错误。
例如，使用 96 kHz 混响器时，请将混响器设为内部时钟，并将控制台输入和输出 AES/EBU 端口的 SRC 设置均设为 SRC In。对于输出端口，请选择 In 和 x2（对应 96k）。
