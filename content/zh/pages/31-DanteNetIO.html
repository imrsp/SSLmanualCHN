网络 I/O 舞台接口箱和设备

SSL Live 控制台可以远程控制以下 SSL 网络 I/O 设备：

  • SB 16.12 - 16 路模拟麦克风/线路输入，8 路模拟输出和 2 对 AES（4 个通道）
  • SB 32.24 - 32 路模拟麦克风/线路输入，16 路模拟输出和 4 对 AES（8 个通道）
  • SB 8.8 - 8 路模拟麦克风/线路输入和输出
  • SB i16 - 16 路模拟麦克风/线路输入
  • A16.D16 - 4 路模拟麦克风/线路输入，12 路模拟线路输入，16 路模拟线路输出，8 对 AES，4 个 GPIO
  • A32 - 32 路模拟线路输入和输出
  • D64 - 32 对 AES
  • GPIO32 - 32 个 GPIO
  • MADI-Bridge - MADI 和 Dante 之间的 64 个通道网桥（48kHz 时）

这些设备也可以由 SSL System T 控制台和用于 PC 的 Network I/O Controller 应用程序控制。

所有网络 I/O 单元都支持 AES67 和 SMPTE 2110-30 AoIP 网络传输标准。

Network I/O Controller 应用程序和所有网络 I/O 设备的手册可以从 www.solidstatelogic.com/live/remote-ecosystem/documents 下载。

控制台支持对 Dante 网络中的 Shure ULX-D 和 Axient Digital 无线麦克风进行控制与监控。请参阅 Shure Wireless Control 部分了解更多详情。

也可以远程控制 Shure ANI 设备的输入参数；请参阅 Shure ANI 部分了解更多详情。

注意：更改网络 I/O 设备的 IP 地址后，请通过断开设备背面的电源线并重新连接来完全重启设备。 使用 Dante Controller 中的 Reboot 按钮不能完全重启设备。

舞台接口箱重置

在新演出使用之前重置 Net I/O 舞台接口箱设置是一个好习惯。可以使用“Reset IO Settings”按钮进行重置。 该按钮位于 Dante I/O 配置页面 (MENU > Setup > I/O > Dante Configuration) 中，点击舞台接口箱并导航到 Setup 选项卡即可找到。

注意：您必须是舞台接口箱的所有者才能重置它。按住“Stagebox Ownership”按钮以取得所有权。

警告：重置舞台接口箱将导致主通道和补偿分离通道的电平跳变。SSL 建议在执行舞台接口箱重置之前，将接收设备上所有受影响的通道静音。

重置 I/O 设置会重置以下设置：

  • 所有所有权状态将设置为无主（参见下面的所有权部分）
  • 所有输入将设置为麦克风模式下的 +35 dB 增益
  • 所有输入的 +48V 将关闭
  • 所有输入的 Pad 将关闭
  • 增益补偿点将重置为 +25 dB（低于默认 +35 dB 麦克风增益设置 10 dB）（参见下面的增益共享部分）
  • 对于 SB 32.24，重置 AES 输入启用按钮，以便将所有输入设置为模拟（如果适用）
  • 所有输入的 AES SRC 将关闭（如果适用）
  • AutoPad 将关闭（如果适用）
  • 所有输入的限幅器将关闭（如果适用）
  • GPO 将关闭（如果适用）
  • 输入和输出静音控制将关闭

IP 地址、网络 A 和 B 链路状态（如果适用）、采样率和工作电平将被保留。

注意：所有参数重置可能需要最多 30 秒，请稍候。

所有权

Net I/O 舞台接口箱可以由 SSL Live 或 System T 控制台以及用于 PC 的 Network I/O Controller 应用程序控制。

控制台系统采用 Ownership（所有权）的概念，为每个舞台接口箱输入和 GPO 指定唯一所有者，以避免控制冲突。 舞台接口箱的常规设置中还有一个所有者，负责控制工作电平和舞台接口箱的其他全局设置（包括 SB 32.24 上的 AES/模拟输入切换）。

因此，在您能够远程控制舞台接口箱之前，您必须先成为该参数的所有者。所有权状态存储在舞台接口箱中。

请注意：

只能控制 Main 舞台接口箱设备。 Comp 设备永远不能被控制或拥有。 有关 Main 和 Comp 设备的更多信息，请参阅使用 Net I/O 舞台接口箱进行增益共享。

首先，确保您要控制的舞台接口箱在“Configured Devices”列表中（参见配置 Dante 设备）。

可执行的所有权操作如下：

  • Stagebox Ownership（舞台接口箱所有权）：更改工作电平，在 AES 和模拟输入之间切换输入源，打开 AES 输入的 SRC，以及打开或关闭 AutoPad
  • Input Ownership（输入所有权）：仅控制输入参数
  • GPO Ownership（GPO所有权）：控制所有 GPO 参数和设置
舞台接口箱所有权

舞台接口箱所有权能够对以下参数进行控制：

  • 工作电平
  • 启用 AES 输入（仅限 SB 32.24）
  • 打开 AES 输入 SRC（仅限 SB 16.12 和 SB 32.24）
  • 打开或关闭 AutoPad（仅限 SB 8.8 或 i16）
  • 启用网络 B SRC（仅限 SB 16.12 和 SB 32.24）
  • 重置舞台接口箱设置

Stagebox Owner 无法控制输入。舞台接口箱的 Stagebox Owner 非必要，如果您不需要调整上述设置，Stagebox Ownership 可以保持无主状态。

要成为 Stagebox owner，请在 Dante 配置页面中点击舞台接口箱，舞台接口箱的详细视图将出现在底部。 按住“Stagebox Ownership”。

Stagebox owner 显示在此按钮左侧，呈青色。

要释放 Stagebox ownership，请再次按住“Stagebox Ownership”以解除。

输入所有权

输入的所有者可以控制所有输入参数，包括增益、Pad、Mic/Line 切换和 +48V 幻象供电开关。

当舞台接口箱输入被路由到控制台上 Channel 通道的输入，并且该输入没有所有者时， 控制台将自动成为该输入的所有者。

但是，如果该输入已被另一台设备拥有，控制台不会自动从该设备夺取所有权。必须手动取得所有权。

Net I/O 舞台接口箱的输入设置可以在通道详细视图的 Input/Routing 页面中查看 （如果该输入已路由到 Channel 通道）或在 Dante 配置页面的舞台接口箱详细视图中查看。

要查看舞台接口箱输入设置，请在 Dante 配置页面中点击舞台接口箱， 然后点击右侧的 Inputs（下图中标注 1）。 控制台拥有的所有输入都将以绿色背景显示，另一台设备拥有的输入则显示为紫色，无主的输入显示为黑色。

点击任意输入（标注 2），然后点击“Edit Settings”（标注 3）可以在弹出的窗口中查看该输入的设置。

以下说明适用于在通道视图或 Dante 配置中查看输入设置。

要取得某路输入的所有权，请进入该输入的设置，您将看到“Ownership”区域，内含一个操作按钮与一个显示框。

“Ownership”区域中的按钮将根据输入的所有权状态更改其功能。

如果当前输入没有所有者，按钮将显示“Own”，框中不会显示任何内容。按住“Own”，您的控制台将成为该输入的所有者。

如果输入已有所有者，按钮将显示“Take”，当前所有者的设备名称将显示在框中。 按住“Take”，您的控制台将接管该输入的所有权。

如果您的控制台已经是所有者，按钮将显示“Release”，您的控制台名称将显示在框中。 按住“Release”，您的控制台将释放所有权，该输入将变为没有所有者。

注意：

接管所有权前无需先释放。

一旦您的控制台拥有所有权，就可以调整增益和输入设置。

如果您的控制台不是输入的所有者，您将看到控件呈灰色，无法进行任何更改。

Network I/O 舞台接口箱的全部输入端口所有权可通过一次操作批量接管或批量释放。请前往 MENU > Setup > I/O > Dante Configuration。 点击舞台接口箱以在其详细视图中查看设置。

在舞台接口箱的 Setup 部分，按住“Inputs Ownership”。有三个选项可用：

Own All 成为所有无主输入的所有者
Take All 成为所有输入的所有者（包括其他设备拥有的输入）
Release All 释放控制台拥有的所有输入的所有权

多个控制设备可以同时拥有同一台舞台接口箱不同输入的所有权，但是同一路输入只能拥有一个所有者，不能共享。

GPO 所有权

一些 Net I/O 设备具有可以从控制台访问的 GPIO 端口。GPO 所有权设置必须从 Dante 配置页面调整。 点击舞台接口箱，打开其详细视图。点击舞台接口箱详细视图右侧的 GPIO。 要成为某个 GPO 的所有者，请根据其当前状态长按“Own”或“Take”。 按住“Rel”即可释放该 GPO 的所有权。

按住“GPO Ownership”，然后选择“Own All”即可成为所有先前无主 GPO 的所有者。

使用“Take All”选项即可取得所有 GPO 的所有权，包括已被其他设备拥有的 GPO。

使用“Release All”选项即可释放控制台拥有的所有 GPO 的所有权。

多个控制设备可以同时拥有同一台舞台接口箱不同 GPO 的所有权，但是同一个 GPO 只能拥有一个所有者，不能共享。

加载演出文件

舞台接口箱设置会保留在 Live 控制台演出文件中，可以通过加载演出文件来调用它们。 保存演出文件时，控制台还会记录下每路输入（以及其他设置）保存时，它是否为其所有者。

加载演出文件时，控制台将尝试重新取得所有保存时由其持有的输入（以及其他设置）所有权。该尝试只在该输入（或其他设置）没有所有者， 或者在加载演出文件时已被控制台所有的情况下才会成功。 如果该输入（或其他设置）为另一台设备拥所有，控制台将无法取得所有权。

但是，在使用“Load with Owner Override”时情况并非如此。该功能允许控制台在加载演出文件时从其他设备接管所有权。请参阅下文此部分。

注意：因此，建议用户在加载演出文件之前，清除他们要使用的舞台接口箱输入所有权。只要这些输入处于无所有者状态或由用户的控制台所有，就可以正确加载演出文件。此时，演出文件的增益设置将按预期应用。

下表显示了在不同演出文件加载场景下舞台接口箱所有权状态的表现。

演出文件加载前的参数所有权状态 演出文件加载后的参数所有权状态 是否将演出文件中的参数设置推送到舞台接口箱？
由控制台拥有 由控制台拥有 是
无所有者 由控制台拥有 是
由另一台设备拥有 仍由另一台设备拥有 否

请注意：

如果在保存演出文件时控制台不是输入（或其他设置）的所有者， 那么当重新加载演出文件时， 控制台也不会尝试成为此输入（或其他设置）的所有者，并且它不会受到演出文件加载的影响。

例如：

一个 L500 是 SB 32.24 上输入 1、2 和 3 的所有者，并且保存了演出文件。

通道 1 所有者 通道 2 所有者 通道 3 所有者
L500 L500 L500

保存演出文件后，一名使用 L200 的操作员取得了输入 1 的所有权。 使用 L500 的操作员释放了输入 2 的所有权，并保留了对输入 3 的所有权。

通道 1 所有者 通道 2 所有者 通道 3 所有者
L200 无 L500

L500 操作员重新加载演出文件，将发生以下情况：

输入 1：L500 尝试成为所有者，但输入 1 由 L200 拥有。 因此 L500 无法取得所有权。任何增益设置、+48V、Pad 等输入设置都不会从 L500 演出文件推送到舞台接口箱的输入 1。输入 1 保留其先前的增益和输入设置。

输入 2：L500 尝试成为所有者。由于输入 2 没有被任何设备所有，因此 L500 成功成为所有者。 全部增益设置、+48V、Pad 等输入设置都将从 L500 演出文件推送到舞台接口箱的输入 2。

输入 3：L500 尝试成为所有者，且输入 3 已由 L500 拥有。因此全部增益设置、+48V、Pad 等输入设置都将从 L500 演出文件推送到舞台接口箱的输入 3。

通道 1 所有者 通道 2 所有者 通道 3 所有者
L200 L500 L500

在加载包含输入设置存储的演出文件之前，请注意舞台接口箱上每个输入当前的所有权状态。 如果其他设备拥有某些输入的所有权，您的控制台将无法在加载演出文件时自动取得这些输入的所有权。

在加载演出文件之前，前往 MENU > Setup > I/O > Dante Configuration，将舞台接口箱添加到“Configured Devices”列表。 点击舞台接口箱，打开其详细视图。要取得全部输入的所有权，请在“Setup”选项卡中按住“Inputs Ownership”按钮并选择“Take All”。

要取得单个输入的所有权，请在“Inputs”选项卡中选择您希望控制的某路输入，然后点击“Edit Settings”。 如果输入由另一台设备（不是您的控制台）拥有，请按住“Take”以取得所有权。如有必要，请事先与当前所有者确认！ 如果输入没有所有者，或者控制台已经是所有者，请继续处理下一个输入。 对您希望控制的全部输入重复此操作，确保在加载演出文件前，需要的输入要么没有所有者，要么所有者为您的控制台。

确保舞台接口箱名称与演出文件中存储的名称匹配。

请注意：

Dante 路由与设备名称相关联，因此更改 Dante 设备名称将删除所有路由。重命名设备应在 Dante Controller 中进行，而不是在控制台上。 在更改设备名称之前，Dante Controller 中会出现一个警告对话框，通知用户音频路由将中断。

然后加载演出文件，所有适当的输入将被拥有，并且设置将从演出文件成功推送到舞台接口箱。

当控制台正常关机时，它将释放其拥有的所有舞台接口箱参数的所有权。

使用所有者覆盖加载演出文件

从 Live 6.1 开始，控制台可以加载演出文件并覆盖现有的网络所有权状态。上文的任何条件都不再适用，并且会发生以下情况：

  • 订阅/路由创建
    • 如果网络中某 Dante 设备的 Rx 通道已有订阅，而演出文件又包含新的订阅，则演出文件将成功建立此订阅。
    • 如果网络上的 Rx 订阅与演出文件建立的订阅不重叠，则它们将保持加载演出文件之前的状态。
  • 舞台接口箱上的参数控制（或任何具有所有权能力的其他设备）
    • 如果音频输入、输出、GPI、GPO 或整个舞台接口箱已被拥有，但此演出文件会在无所有者时取得所有权，则使用此模式加载将取得所有权。
    • 如果网络上的某些输入/设备由另一台设备拥有，但演出文件中未包含对这些输入/设备的所有权，则所有权状态将保持加载演出文件之前的状态。

控制

点击 Net I/O 舞台接口箱设备时，所有参数和设置将显示在 Dante Configuration 的详细视图中。

点击“Inputs”或“Outputs”将显示每个输入/输出的设置。

点击输入可控制输入参数〔前提是拥有该输入的所有权（参见上面的“所有权”部分）〕。

当来自 SSL Network I/O 舞台接口箱的输入被路由到通道的输入时，输入参数也将显示在详细视图的 Input/Routing 部分。 然后可以通过快速控制、控制区块和 TaCo 应用程序控制这些参数。

SB 32.24 模拟与 AES 输入

在 96 kHz 下运行的 Net I/O SB 32.24 舞台接口箱最多有 32 个可用输入。输入可以是模拟（最多 32 个）或 AES（最多 4 对；8 个通道）。用户可以按对进行定义。

要在模拟和 AES 输入之间进行选择，请前往 Dante 配置页面，点击 SB 32.24 Main 设备以调出设备的详细视图。

按住 Stagebox Ownership 以成为 Stagebox 的所有者。

点击 AES I/O ，然后点击 AES 对。

按住 AES Input Enabled 按钮以启用 AES 输入并禁用相应的模拟输入对。 舞台接口箱正面上对应的 AES IN ACT LED 将亮起绿色，表示 AES 端口处于工作状态。

在 48 kHz 下，SB 32.24 的所有模拟和 AES 输入可同时使用。

使用 Net I/O 舞台接口箱进行增益共享

什么是增益补偿输出？

SSL Net I/O 舞台接口箱为每个话筒放大器输入提供两路 Dante 输出；一路是主输出，另一路是增益补偿输出，提供恒定的信号电平。

当两个控制台需要接收相同的麦克风放大器输入时，增益补偿输出非常有用。以典型的 FOH 与 Monitor 双台方案为例：，FOH 控制台接收主输出，监听控制台接收增益补偿输出。这样，FOH 调音师调整话放增益时，不会影响监听侧的接收电平信号。

从舞台接口箱的“Compensated output”接收信号的设备可以是任何支持 Dante 的设备（它们不必是 SSL 设备）。

增益补偿输出如何工作？

模拟话筒放大器电路的输出馈送到 AD 转换器，模拟信号转为数字信号后被馈送到两条路径。 第一条路径将数字音频路由到 Dante 主输出。 第二条路径将对信号进行数字增益控制，它会自动对话放增益所做的任何电平变化施加一个大小相等、方向相反的增益偏移，从而实现增益补偿，使输出端电平保持恒定。

例如，如果话放电平增加了 3 dB，补偿输出的数字增益控制将减少 3 dB 以保持恒定的信号电平。

Main 和 Comp 设备

当 SSL Network I/O 舞台接口箱连接到 Dante 网络时，它将在控制台上显示为两个独立的 Dante 设备：

  • Main uncompensated/gain dependant 设备
  • Comp gain compensated 设备

注意：只有 SB 16.12 和 SB 32.24 舞台接口箱会在 Dante Controller 中显示为两个独立的设备。其他 Net I/O 设备在 Dante Controller 中将显示为单个设备。

Main 设备接收未补偿/随话放增益变化的信号，并控制舞台接口箱。

Comp 设备接收增益补偿的信号，并且不会拥有控制权。

选定网络中哪个控制台负责接收未补偿信号并需要增益控制。 此控制台需要从 Main 舞台接口箱设备进行路由。若要启用增益共享，该设备必须为 SSL 控制台。

选定哪些 Dante 设备应接收增益补偿信号。这些设备将需要从 Comp 舞台接口箱设备进行路由。

当 Main 设备更改增益时，Main 设备上的信号电平将根据所应用的增益而变化。 Comp 设备上的信号电平不会改变，因为它会进行适当的增益补偿。

调整校准点

在舞台接口箱输入的增益推子上有一个 γ 符号。这表示数字增益补偿或校准点的参考点。

当增益推子移动到校准点上方或下方时， Comp 设备上对应的输入将根据增益值与校准点位置之间的差值进行微调。

例如，如果主控制台将增益设置为高于 γ 点 3 dB，则Comp 设备会进行 -3 dB 的数字微调，这样一来， Comp 设备上的信号电平保持不变。 如果主控制台将增益精确设置在 γ 点，则不会对 Comp 设备应用增益补偿。

Main 设备控制校准点。Main 设备可以重置校准点或手动调整它。

重新校准会重置 γ 点，从而重置增益跟踪点。只有在粗略增益水平已经设置好之后，才应使用重新校准。

当输入处于 Line 模式时，重新校准会将 γ 点移动到模拟增益推子的确切位置。

当输入处于 Mic 模式时，重新校准会将 γ 点移动到模拟增益推子位置下方 10 dB 处。 这样做的原因是麦克风电平信号可能比线路电平信号的动态更大，并且需要添加更多增益，这一操作旨在预留余量并确保补偿输入不会因模拟增益大幅变动而产生过量数字修正，导致数字过载。

可以使用 + 或 - 3 dB 按钮以 3 dB 的步长更紧密地对齐校准点。下次输入增益重新校准时，此操作将被重置。

实际应用

增益共享的主要实际应用是在 FOH 控制台和 Monitor 控制台之间共享 I/O 时。

其中一个控制台拥有输入的所有权，因此可以控制增益并接收依赖增益的信号。这必须是 SSL 控制台。

另一个控制台从增益补偿的分路输出接收信号。这不一定是 SSL 控制台，可以是任何支持 Dante 的设备。

拥有舞台接口箱输入所有权/控制权的控制台将设置系统的增益结构。在更改校准点之前，应先由此控台将增益粗调到可用的安全工作值。

系统的增益结构设置完毕后，应在每个输入上按下 Recalibrate 按钮。这将为增益补偿创建一个参考点或校准点。

在按下 Recalibrate 按钮之前，补偿输入不会设置其增益。

可以一次性重新校准特定设备的所有已拥有输入。前往 Dante I/O 配置页面 (MENU > Setup > I/O > Dante Configuration) 并点击 Main 舞台接口箱设备。 现在按住 Recalibrate All Owned Inputs。

可以使用 IO Functions > Recalibrate All User Key 重新校准所有已拥有输入（跨所有设备）。

如果在演出前某个特定输入的增益结构发生巨大变化，可以再次使用重新校准来重置校准点。

警告：

在演出期间不要在打开的通道上使用重新校准， 因为这会导致 Comp 设备上的增益发生变化。

SSL 建议在按下重新校准之前，将任何接收增益补偿信号的设备上的所有输出静音， 因为这可能会导致增益跳变。

Net I/O SB 32.24 和 SB 16.12 舞台接口箱网络 B 连接上的 SRC

SB 32.24 和 SB 16.12 舞台接口箱的网络 B 上提供了一个采样率转换器 (SRC)，允许增益补偿的分路输出以不同的采样率或在不同的时钟域中运行。

使用设备“Setup”页面中的“Net B SRC In”按钮来启用 SRC（您必须是设备所有者才能启用此按钮）。 使用 Dante Controller 设置网络 B 模块的采样率。

示例：用于演出的 FOH 现场控制台也用于广播。现场控制台和整个 I/O 基础设施可以以 96 kHz 运行，以获得最佳音频质量和低延迟。 可以将 Net B 分路输出连接采样率转换为 48 kHz，以馈送给广播车。
